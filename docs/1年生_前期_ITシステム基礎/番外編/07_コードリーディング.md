## 読むファイルの順番

```
① models.py     ← OOP（クラス）の集大成
② schemas.py    ← OOP（クラス）の集大成
③ main.py       ← FastAPI（デコレータ）の集大成
④ routers/tasks.py ← OOP + FastAPI 両方が登場
⑤ database.py   ← 接続設定（参考）
```

## ① models.py を読む

```python
from sqlalchemy import Column, Integer, String, Boolean
from database import Base

class Task(Base):
    __tablename__ = "tasks"

    id    = Column(Integer, primary_key=True, index=True)
    title = Column(String(200), nullable=False)
    done  = Column(Boolean, default=False)
```

### 02_OOP入門 との対応

**`class Task` → クラスの定義（02 ステップ1 で学んだ）**

```
02 で学んだ書き方         実際のコード
─────────────────────    ──────────────────────
class Task:              class Task(Base):
    pass                     __tablename__ = "tasks"
```

`(Base)` は **継承** といいます。`Base` というクラスの機能を引き継いで、`Task` クラスを作っています。
「SQLAlchemy の機能を全部使えるようにする設定」と理解してください。

---

**`id / title / done` → クラス変数（インスタンスが持つデータ）**

02 の `Task` クラスでは `self.title` や `self.done` をコンストラクタで定義しました。
`models.py` では SQLAlchemy の `Column()` を使って同じことをしています。

```
02 で学んだ書き方               実際のコード（models.py）
─────────────────────────    ──────────────────────────────────
class Task:                  class Task(Base):
    def __init__(self):
        self.title = ...         title = Column(String(200))
        self.done = False        done  = Column(Boolean, default=False)
```

書き方は違いますが、「**クラスがどんなデータを持つか**」を定義している点は同じです。

---

**インスタンスを作る場面（routers/tasks.py より）**

```python
# Task クラスからインスタンスを作る（02 のステップ2 と同じ！）
new_task = Task(title=task_in.title)
#          ↑ Task クラス   ↑ title に値を渡す
```

02 の練習と比較：
```python
# 02 の練習                    実際のコード
task1 = Task("買い物に行く")   new_task = Task(title="買い物に行く")
```

やっていることは全く同じです。

## ② schemas.py を読む

```python
from pydantic import BaseModel

class TaskCreate(BaseModel):
    title: str

class TaskResponse(BaseModel):
    id: int
    title: str
    done: bool

    class Config:
        from_attributes = True
```

### 02_OOP入門 との対応

**2 つのクラスが定義されている → 用途が違う**

```
クラス名         役割                     インスタンスが作られるタイミング
────────────    ─────────────────────    ──────────────────────────────
TaskCreate      POST リクエストを受け取る  ブラウザから「追加」が押されたとき
TaskResponse    レスポンスを返す           API が結果を返すとき
```

**`title: str` → 型アノテーション付きのクラス変数**

02 では `self.title = title` のように書きましたが、
Pydantic クラスでは `title: str` と書くだけで「title は文字列」と定義できます。

```python
# 02 の書き方（通常のクラス）       schemas.py の書き方（Pydantic クラス）
def __init__(self, title):         class TaskCreate(BaseModel):
    self.title = title                 title: str   ← 1行で書ける！
```

**`class Config` → クラスの中にクラス**

クラスの中に別のクラスを書くことができます。
ここでは SQLAlchemy のモデルオブジェクトをそのまま渡せるようにする設定を書いています。

## ③ main.py を読む

```python
from fastapi import FastAPI
from fastapi.staticfiles import StaticFiles
from fastapi.responses import FileResponse
from database import engine, Base
import models
from routers import tasks

Base.metadata.create_all(bind=engine)

app = FastAPI()

app.include_router(tasks.router)

app.mount("/static", StaticFiles(directory="static"), name="static")

@app.get("/")
def read_index():
    return FileResponse("static/index.html")
```

### 03_FastAPI入門 との対応

**`app = FastAPI()` → FastAPI のインスタンスを作る（03 で学んだ）**

```
03 で学んだ                実際のコード
─────────────────────    ──────────────────────
app = FastAPI()           app = FastAPI()   ← 全く同じ！
```

---

**`@app.get("/")` → デコレータ（03 で学んだ）**

```python
@app.get("/")          # ← 「/ に GET リクエストが来たらこの関数を呼ぶ」
def read_index():
    return FileResponse("static/index.html")
```

03 の Hello World との比較：
```
03 の練習                            実際のコード
─────────────────────────────────   ──────────────────────────────────
@app.get("/")                        @app.get("/")
def read_root():                     def read_index():
    return {"message": "Hello!"}         return FileResponse("static/index.html")
```

`return` の内容が違うだけで、デコレータの使い方は全く同じです。

---

**`app.include_router(tasks.router)` → 新登場：ルーター**

03 では全てのエンドポイントを `main.py` に書いていました。
実際のシステムでは、タスク関連の API は `routers/tasks.py` にまとめて書き、
`main.py` で「このルーターを使います」と登録します。

```
03 の骨格（main.py に全部書く）       実際のシステム（ファイルを分ける）
─────────────────────────────────   ──────────────────────────────────
@app.get("/tasks")                   routers/tasks.py に書く
@app.post("/tasks")                       ↓
@app.put("/tasks/{id}")              main.py で登録する
@app.delete("/tasks/{id}")           app.include_router(tasks.router)
```

**なぜファイルを分けるのか？**
→ タスク・ユーザー・商品など機能が増えるとき、1 ファイルにまとめると読みにくくなるため。

---

**`Base.metadata.create_all(bind=engine)` → テーブルの自動作成**

アプリが起動するたびに「tasks テーブルがなければ作る」を自動で行います。
`models.py` の `Task` クラスを元にテーブルを作ります。

## ④ routers/tasks.py を読む

OOP（02）と FastAPI（03）の両方が登場する最重要ファイルです。

```python
from fastapi import APIRouter, Depends, HTTPException
from sqlalchemy.orm import Session
from database import get_db
from models import Task
from schemas import TaskCreate, TaskResponse

router = APIRouter(prefix="/tasks", tags=["tasks"])
```

**`router = APIRouter(...)` → app と同じ、ルーター専用のインスタンス**

```
main.py             routers/tasks.py
────────────────    ──────────────────────────────
app = FastAPI()     router = APIRouter(prefix="/tasks")
@app.get("/")       @router.get("")
```

`prefix="/tasks"` の意味：このルーターのデコレータに書く URL の前に `/tasks` が自動でつきます。

```
@router.get("")   →  実際の URL は /tasks
@router.get("/{task_id}")  →  実際の URL は /tasks/{task_id}
```

---

### GET /tasks — 一覧取得

```python
@router.get("", response_model=list[TaskResponse])
def get_tasks(db: Session = Depends(get_db)):
    return db.query(Task).all()
```

**`@router.get("")` → デコレータ（03 で学んだ）**
`router` に付けることで `/tasks` への GET リクエストに対応します。

**`response_model=list[TaskResponse]` → レスポンスの形を指定**
`schemas.py` の `TaskResponse` クラスのリストを返すと宣言しています。

**`Depends(get_db)` → 新登場：依存性注入**
「`get_db` 関数の結果（DB への接続）を `db` に渡してください」という FastAPI の仕組みです。
API が呼ばれるたびに自動で DB との接続を準備してくれます。

**`db.query(Task).all()` → Task クラスのインスタンスを全件取得**
```
db.query(Task).all()
             ↑
         OOP の Task クラスを指定
         → tasks テーブルの全行を Task インスタンスのリストとして取得
```

---

### POST /tasks — タスク作成

```python
@router.post("", response_model=TaskResponse)
def create_task(task_in: TaskCreate, db: Session = Depends(get_db)):
    new_task = Task(title=task_in.title)  # ← OOP！インスタンスを作る
    db.add(new_task)
    db.commit()
    db.refresh(new_task)
    return new_task
```

**`task_in: TaskCreate` → schemas.py のクラスがここに登場**
ブラウザから送られてきた JSON が自動で `TaskCreate` インスタンスに変換されます。

```
ブラウザが送るデータ        FastAPI が自動変換
────────────────────    ──────────────────────────────
{"title": "買い物"}  →  task_in = TaskCreate(title="買い物")
                                             ↑ task_in.title で取り出せる
```

**`new_task = Task(title=task_in.title)` → OOP（02 ステップ2 と同じ）**

```
02 の練習                         実際のコード
─────────────────────────────    ──────────────────────────────
task1 = Task("買い物に行く")       new_task = Task(title=task_in.title)
```

インスタンスを作ってデータを保持させる、まさに OOP です。

---

### PUT /tasks/{task_id} — 完了切り替え

```python
@router.put("/{task_id}", response_model=TaskResponse)
def toggle_task(task_id: int, db: Session = Depends(get_db)):
    task = db.query(Task).filter(Task.id == task_id).first()

    if task is None:
        raise HTTPException(status_code=404, detail="タスクが見つかりません")

    task.done = not task.done   # ← OOP！インスタンスのデータを変更
    db.commit()
    db.refresh(task)
    return task
```

**`/{task_id}` → パスパラメータ（03 で学んだ）**

```
03 の練習                          実際のコード
─────────────────────────────     ──────────────────────────────
@app.get("/tasks/{task_id}")       @router.put("/{task_id}")
def get_task(task_id: int):        def toggle_task(task_id: int, ...):
```

URL に含まれる数字（例：`/tasks/3`）が `task_id = 3` として渡ってきます。

**`task.done = not task.done` → インスタンスのデータを直接変更**

```
取得した task インスタンスの done を反転させる
task.done が False → not False → True  （未完了 → 完了）
task.done が True  → not True  → False （完了 → 未完了）
```

---

### DELETE /tasks/{task_id} — 削除

```python
@router.delete("/{task_id}")
def delete_task(task_id: int, db: Session = Depends(get_db)):
    task = db.query(Task).filter(Task.id == task_id).first()

    if task is None:
        raise HTTPException(status_code=404, detail="タスクが見つかりません")

    db.delete(task)
    db.commit()
    return {"message": f"タスク '{task.title}' を削除しました"}
```

**`raise HTTPException(status_code=404, ...)` → エラーを返す**

`task` が見つからなかったとき（`is None`）、404 エラーをブラウザに返します。
`raise` は「このエラーを今すぐ発生させる」という Python の構文です。

## ⑤ 全体のつながり

```
main.py
  │ app = FastAPI()          ← FastAPI のインスタンス（03）
  │ app.include_router(...)  ← ルーターを登録
  │ @app.get("/")            ← デコレータ（03）
  │
  ├── routers/tasks.py
  │     │ router = APIRouter(...)
  │     │ @router.get / post / put / delete  ← デコレータ（03）
  │     │ Task(title=...)                    ← インスタンス作成（02）
  │     │ task.done = not task.done          ← インスタンス操作（02）
  │     │
  │     ├── models.py
  │     │     class Task(Base)   ← クラス定義（02）
  │     │     id / title / done  ← クラス変数（02）
  │     │
  │     ├── schemas.py
  │     │     class TaskCreate(BaseModel)    ← クラス定義（02）
  │     │     class TaskResponse(BaseModel)  ← クラス定義（02）
  │     │
  │     └── database.py
  │           DATABASE_URL / engine / get_db  ← DB 接続設定
  │
  └── static/index.html
        fetch("/tasks", ...)  ← JavaScript から API を呼び出す（06）
```

## まとめ：02・03 で学んだことの使われ方

| 学んだ内容 | 資料 | 実際のコード |
|-----------|------|-------------|
| `class クラス名:` でクラスを定義 | 02 | `models.py` の `Task`、`schemas.py` の `TaskCreate` など |
| `インスタンス = クラス名(引数)` | 02 | `new_task = Task(title=...)` |
| インスタンスのデータを変更 | 02 | `task.done = not task.done` |
| `app = FastAPI()` | 03 | `main.py` の 1 行目 |
| `@app.get(...)` デコレータ | 03 | `@router.get("") / @router.post("") など` |
| `/{パラメータ}` パスパラメータ | 03 | `@router.put("/{task_id}")` |
