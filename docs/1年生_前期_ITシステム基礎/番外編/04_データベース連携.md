# データベース連携（SQLAlchemy + MySQL）

## 今回のゴール
- SQLAlchemy の役割を理解する
- `database.py` で MySQL への接続設定ができる
- `models.py` でテーブルをクラスとして定義できる

---

## 1. SQLAlchemy とは？

**SQLAlchemy = Python から DB を操作するためのライブラリ（ORM）**

**ORM（Object-Relational Mapping）** とは、
**テーブルの行 ↔ Python のインスタンス** と対応させる仕組みです。

```
普通の SQL（文字列で書く）     SQLAlchemy（Python のクラスで書く）
─────────────────────────    ──────────────────────────────────
SELECT * FROM tasks;          db.query(Task).all()
INSERT INTO tasks (title)     task = Task(title="買い物")
  VALUES ('買い物');           db.add(task)
                              db.commit()
```

SQL をそのまま書く代わりに、**Python のコードで DB を操作できる**ようになります。
コードが読みやすくなり、ミスも減ります。

## 2. database.py — DB 接続の設定

```python
# database.py

from sqlalchemy import create_engine
from sqlalchemy.ext.declarative import declarative_base
from sqlalchemy.orm import sessionmaker

# ① 接続文字列（どの DB に、誰が、何というパスワードで接続するか）
#    書き方: "mysql+pymysql://ユーザー名:パスワード@ホスト/DB名"

# Windows（XAMPP）の場合 → デフォルトはパスワードなし
DATABASE_URL = "mysql+pymysql://root:@localhost/todo_app"

# Mac（Homebrew）の場合 → インストール時に設定したパスワードを入れる
# DATABASE_URL = "mysql+pymysql://root:あなたのパスワード@localhost/todo_app"

# ② エンジン = DB との接続を管理するオブジェクト
engine = create_engine(DATABASE_URL)

# ③ セッション = DB への「会話の窓口」を作る設定
#    autocommit=False → 明示的に commit() するまで保存しない
#    autoflush=False  → 自動で flush（DB への即時反映）しない
SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)

# ④ モデルクラスの基底クラス（models.py で使う）
Base = declarative_base()
```

### DB セッションを取得する関数

```python
# database.py に追加

def get_db():
    """API が呼ばれるたびに DB セッションを作り、終わったら閉じる"""
    db = SessionLocal()
    try:
        yield db          # ← API 関数に db を渡す
    finally:
        db.close()        # ← 処理が終わったら必ず閉じる
```

## 3. models.py — テーブルをクラスで定義する

ここが **OOP（オブジェクト指向）と DB が交わるポイント**です。

```python
# models.py

from sqlalchemy import Column, Integer, String, Boolean
from database import Base  # database.py の Base を使う

class Task(Base):
    """
    tasks テーブルの定義
    クラス = テーブル
    クラスのインスタンス = テーブルの 1 行
    """
    __tablename__ = "tasks"  # MySQL のテーブル名

    # カラムの定義（Column = 列）
    id    = Column(Integer, primary_key=True, index=True)
    title = Column(String(200), nullable=False)  # NULL 禁止
    done  = Column(Boolean, default=False)        # デフォルトは未完了
```

### クラスとテーブルの対応

```
Python クラス                 MySQL テーブル（tasks）
─────────────────────────    ──────────────────────
class Task(Base):            CREATE TABLE tasks (
    __tablename__ = "tasks"
    id    = Column(Integer)      id    INT PRIMARY KEY,
    title = Column(String(200))  title VARCHAR(200),
    done  = Column(Boolean)      done  BOOLEAN
                             );

Task(title="買い物")         INSERT INTO tasks (title) VALUES ('買い物');
→ インスタンス = 1 行
```

## 4. テーブルを自動作成する

モデルを定義したら、`main.py` に以下を追加するとテーブルが自動作成されます。

```python
# main.py の上部に追加

from database import engine, Base
import models  # models.py を読み込む

# アプリ起動時にテーブルを作成（なければ作る、あればそのまま）
Base.metadata.create_all(bind=engine)
```

起動して MySQL でテーブルが作られたか確認します。

**Windows（XAMPP）：phpMyAdmin で確認**

1. ブラウザで `http://localhost/phpmyadmin` を開く
2. 左メニューの `todo_app` をクリック
3. `tasks` テーブルが表示されていれば OK

**Mac（ターミナルで確認）：**

```bash
mysql -u root -p
```

```sql
USE todo_app;
SHOW TABLES;
-- tasks テーブルが作られていれば OK

DESCRIBE tasks;
-- +-------+--------------+------+-----+---------+
-- | Field | Type         | Null | Key | Default |
-- +-------+--------------+------+-----+---------+
-- | id    | int          | NO   | PRI | NULL    |
-- | title | varchar(200) | NO   |     | NULL    |
-- | done  | tinyint(1)   | YES  |     | 0       |
-- +-------+--------------+------+-----+---------+

EXIT;
```

## 6. ポイント整理

```
① database.py
   - DATABASE_URL でどの MySQL に接続するかを設定
   - SessionLocal で「DB との会話の窓口」を作る準備
   - get_db() で API が DB を使えるようにする

② models.py
   - class Task(Base) でテーブルをクラスとして定義
   - Column() でカラム（列）を定義
   - インスタンス 1 つ = テーブルの 1 行

③ main.py
   - Base.metadata.create_all() でテーブルを自動作成
```
