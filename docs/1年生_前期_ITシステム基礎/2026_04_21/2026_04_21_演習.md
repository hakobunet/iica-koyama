## 総合演習：自動販売機プログラムの開発

### 【STEP 1】変数と定数の基本

**要件:**
- 商品価格を定数 `PRICE`（150）で定義する。
- 標準入力から「投入金額」を数値として受け取る。
- 投入金額から価格を引いた「残金」を計算し、`Remaining: [数値]` と出力する。

JavaScript

```
const reader = require('readline').createInterface({
  input: process.stdin,
  output: process.stdout
});

const lines = [];
reader.on('line', (line) => {
  lines.push(line);
});

reader.on('close', () => {
  const PRICE = 150;
  let money = Number(lines[0]); // 1行目の入力
  
  let remaining = money - PRICE;
  console.log(`Remaining: ${remaining}`);
});
```

### 【STEP 2】if文による条件分岐

**要件:**
- 投入金額が `PRICE` 未満の場合、`Error: Insufficient funds` と出力して終了する。
- 足りている場合のみ、お釣りを出力する。

JavaScript

```
reader.on('close', () => {
  const PRICE = 150;
  let money = Number(lines[0]);

  if (money < PRICE) {
    console.log("Error: Insufficient funds");
  } else {
    let remaining = money - PRICE;
    console.log(`Remaining: ${remaining}`);
  }
});
```

### 【STEP 3】配列とfor文（複数購入）

**要件:**
- 入力の1行目に「投入金額」、2行目に「購入したい本数」が与えられる。
- 1本買うごとに残金を計算し、その履歴を配列 `history` に追加していく。
- 購入途中で残金が足りなくなったら、そこでループを中断する。
- 最後に配列の中身をループで順番に出力する。

JavaScript

```
reader.on('close', () => {
  let money = Number(lines[0]);
  const count = Number(lines[1]);
  const PRICE = 150;
  const history = [];

  // 購入処理のループ
  for (let i = 0; i < count; i++) {
    if (money >= PRICE) {
      money -= PRICE;
      history.push(money);
    } else {
      break;
    }
  }

  // 配列の中身をループで出力（for文バージョン）
  for (let i = 0; i < history.length; i++) {
    // history[i] で配列の各要素にアクセスします
    console.log(`Step ${i + 1}: ${history[i]}`);
  }
});
```


```
reader.on('close', () => {
  let money = Number(lines[0]);
  const count = Number(lines[1]);
  const PRICE = 150;
  const history = [];

  for (let i = 0; i < count; i++) {
    if (money >= PRICE) {
      money -= PRICE;
      history.push(money);
    } else {
      break;
    }
  }

  // 配列の中身をループで出力
  history.forEach((rem, index) => {
    console.log(`Step ${index + 1}: ${rem}`);
  });
});
```

### 【STEP 4】リファクタリング：switch文による多分岐

**要件:**

- 入力の1行目に「投入金額」、2行目に「商品名（Coke, Water, Coffee）」が与えられる。
- `switch` 文を使い、商品名によって価格を切り替える。
    - Coke: 150, Water: 100, Coffee: 120
    - それ以外: `Error: Unknown item` と出力。

JavaScript

```
reader.on('close', () => {
  let money = Number(lines[0]);
  const itemName = lines[1]; // 商品名
  let price = 0;

  // if / else if を使った多分岐
  if (itemName === "Coke") {
    price = 150;
  } else if (itemName === "Water") {
    price = 100;
  } else if (itemName === "Coffee") {
    price = 120;
  } else {
    // どの条件にも当てはまらない場合（switchのdefaultに相当）
    console.log("Error: Unknown item");
    return; 
  }

  // 購入判定
  if (money >= price) {
    console.log(`Remaining: ${money - price}`);
  } else {
    console.log("Error: Insufficient funds");
  }
});
```

```
reader.on('close', () => {
  let money = Number(lines[0]);
  const itemName = lines[1]; // 2行目：商品名
  let price = 0;

  switch (itemName) {
    case "Coke":
      price = 150;
      break;
    case "Water":
      price = 100;
      break;
    case "Coffee":
      price = 120;
      break;
    default:
      console.log("Error: Unknown item");
      return; // 処理を終了
  }

  if (money >= price) {
    console.log(`Remaining: ${money - price}`);
  } else {
    console.log("Error: Insufficient funds");
  }
});
```

### 【STEP 5】最終リファクタリング：関数化

### 【STEP 5】リファクタリング：関数化とテンプレートリテラル

**要件:**
- 商品名から価格を返す関数 `getPrice(name)` を作成する。
- 購入後の残金を計算して文字列を返す関数 `createMessage(remaining)` を作成する。
- メイン処理をこれらの関数を使って整理する。

JavaScript

```
// 価格を判定する関数
const getPrice = (name) => {
  switch (name) {
    case "Coke":   return 150;
    case "Water":  return 100;
    case "Coffee": return 120;
    default:       return null;
  }
};

// メッセージを作る関数（テンプレートリテラル）
const createMessage = (remaining) => `Result: Remaining balance is ${remaining} yen.`;

reader.on('close', () => {
  const money = Number(lines[0]);
  const itemName = lines[1];
  
  const price = getPrice(itemName);

  if (price === null) {
    console.log("Error: Unknown item");
  } else if (money < price) {
    console.log("Error: Insufficient funds");
  } else {
    const remaining = money - price;
    console.log(createMessage(remaining));
  }
});
```

### 💡 授業での指導ポイント

- **非同期処理の意識:** `reader.on('close', ...)` の外に処理を書くと、入力が完了する前に実行されてしまいエラーになることを伝えてください。
    
- **配列の添字:** `lines[0]` や `lines[1]` など、入力がどの順番で入ってくるかを意識させることが大切です。
    
- **trim() の必要性:** 今回のコードには入れていませんが、入力に余計な空白が含まれる場合があるため、実戦では `lines.push(line.trim());` とすることを教えるとより親切です。
