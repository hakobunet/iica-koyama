curl だけでなく、ブラウザ上でトークン認証の動作を視覚的に確認できるページを用意します。

### 概要

`/api-test` にアクセスするとログイン画面が開きます。  
ログインすると、API で取得した生徒一覧が表示され、追加・削除も操作できます。

> 追加するのは `routes/web.php` の1行と、ビューファイル1つだけです。

### Step 1｜`routes/web.php` にルートを追加する

VSCode でプロジェクトの `routes/web.php` を開きます。

```
（リポジトリ名）/
└── routes/
    └── web.php   ← これを開く
```

ファイルの末尾に以下の1行を追加します：

```php
Route::get('/api-test', fn() => view('api-test'));
```

> **`use` の追加は不要です。**  
> このルートはクロージャ（無名関数）を使っているため、コントローラのクラスを呼び出しません。  
> `use App\Http\Controllers\...` の行を追加しなくても動作します。

追加後の `routes/web.php` はこのような形になります：

```php
<?php

use Illuminate\Support\Facades\Route;

Route::get('/', function () {
    return view('welcome');
});

Route::get('/api-test', fn() => view('api-test'));  // ← この行を追加
```

**追加したら Ctrl+S（Mac は Cmd+S）で保存します。**

// ─── UI 更新 ─────────────────────────────────────────────
function updateUI() {
    const loggedIn = !!token;

        document.getElementById('token-display').textContent  = token;
    }

    document.getElementById('btn-with-token').disabled = !loggedIn;
    document.getElementById('btn-me').disabled         = !loggedIn;

// ─── レスポンス表示 ──────────────────────────────────────
function showResult(status, data, endpoint, method = 'GET') {
    // ステータスバッジ
    const pill = document.getElementById('res-status');
    const statusText = status === 0 ? 'ERROR' : String(status);
    pill.textContent = statusText;
    pill.className = 'status-pill ' + (
        status === 200 ? 'status-200' :
        status === 201 ? 'status-201' :
        status === 401 ? 'status-401' :
        status === 422 ? 'status-422' :
        status >= 400  ? 'status-4xx' :
        status >= 500  ? 'status-5xx' : 'status-0'
    );

    document.getElementById('res-endpoint').textContent = `${method} ${endpoint}`;

    // JSON シンタックスハイライト
    const json = JSON.stringify(data, null, 2);
    document.getElementById('res-body').innerHTML = highlight(json);
    document.getElementById('res-body').classList.remove('empty');

    // ヒントメッセージ
    const hint = document.getElementById('hint');
    if (status === 401) {
        hint.className = 'hint-box warn';
        hint.innerHTML = '🔒 <strong>401 Unauthenticated</strong><br>トークンがない（または無効な）リクエストは拒否されます。<br>ログインしてトークンを取得してから再度試してください。';
    } else if (status === 200 && data.data) {
        hint.className = 'hint-box';
        hint.innerHTML = '✅ <strong>200 OK</strong><br>有効なトークンを Bearer ヘッダーに付けたリクエストが成功しました。<br>React からも同じヘッダーを付けてデータを取得します。';
    } else if (status === 200 && data.token) {
        hint.className = 'hint-box';
        hint.innerHTML = '✅ <strong>ログイン成功</strong><br>トークンが発行されました。このトークンを <code>Authorization: Bearer &lt;token&gt;</code> ヘッダーに付けてAPIを呼びます。<br>右のパネルの <code>token</code> フィールドをコピーしてcurlでも使えます。';
    } else if (status === 200 && data.message === 'Logged out') {
        hint.className = 'hint-box warn';
        hint.innerHTML = '🔓 <strong>ログアウト成功</strong><br>サーバー側でトークンが削除されました。<br>このトークンは無効になり、以降のリクエストで 401 が返ります。';
    } else {
        hint.className = 'hint-box info';
        hint.innerHTML = `レスポンス受信：HTTP ${status}`;
    }
}

// ─── API 呼び出し ─────────────────────────────────────────
async function doLogin() {
    const email    = document.getElementById('email').value;
    const password = document.getElementById('password').value;

    const res = await fetch('/api/login', {
        method: 'POST',
        headers: {
            'Content-Type': 'application/json',
            'Accept':       'application/json',
        },
        body: JSON.stringify({ email, password }),
    });
    const data = await res.json();
    showResult(res.status, data, '/api/login', 'POST');

    if (res.ok && data.token) {
        token    = data.token;
        userName = data.user?.name || 'ユーザー';
        localStorage.setItem('api_token',    token);
        localStorage.setItem('api_username', userName);
        updateUI();
    }
}

async function doLogout() {
    const res = await fetch('/api/logout', {
        method: 'POST',
        headers: {
            'Accept':        'application/json',
            'Authorization': `Bearer ${token}`,
        },
    });
    const data = await res.json();
    showResult(res.status, data, '/api/logout', 'POST');

    token    = null;
    userName = null;
    localStorage.removeItem('api_token');
    localStorage.removeItem('api_username');
    updateUI();
}

// ─── JSON シンタックスハイライト ──────────────────────────
function highlight(json) {
    return json
        .replace(/&/g, '&amp;').replace(/</g, '&lt;').replace(/>/g, '&gt;')
        .replace(
            /("(\\u[a-zA-Z0-9]{4}|\\[^u]|[^\\"])*"(\s*:)?|\b(true|false|null)\b|-?\d+(?:\.\d*)?(?:[eE][+\-]?\d+)?)/g,
            match => {
                if (/^"/.test(match)) {
                    return /:$/.test(match)
                        ? `<span class="jk">${match}</span>`
                        : `<span class="js">${match}</span>`;
                }
                if (/true|false|null/.test(match)) return `<span class="jb">${match}</span>`;
                return `<span class="jn">${match}</span>`;
            }
        );
}

### Step 4｜動作を確認する

#### ① ログインする

> ```
> `exit` で tinker を終了したら、この手順に戻ってください。

1. 入力済みのメールアドレスとパスワードを確認する
2. 「ログイン（POST /api/login）」ボタンを押す
3. ヘッダーに「ログイン中：テストユーザー」と表示され、生徒一覧テーブルが表示されれば成功

ヘッダー右側に発行されたトークンが短縮表示されています。  
このトークンが、以降の全リクエストに自動的に付与されます。

> **ページを閉じても大丈夫です。**  
> トークンはブラウザの `localStorage` に保存されるため、再度開いてもログイン状態が維持されます。

---

#### ② 生徒一覧を確認する

ログイン後すぐに生徒一覧が表示されます。  
テーブルの上に「`GET /api/students` → `HTTP 200`」と表示されていることを確認します。

> **ログインしていないとこのデータは取得できません。**  
> トークンなしで `GET /api/students` を叩くと `401 Unauthenticated` が返ります。  
> ログイン → 一覧表示 という流れが、トークン認証の動作そのものです。

---

#### ③ 生徒を追加する

「新規追加（POST /api/students）」フォームに入力して「追加」ボタンを押します。

| 項目      | 入力例                       |
| ------- | ------------------------- |
| 名前      | `新しい生徒`                   |
| メールアドレス | `new-student@example.com` |
| 点数      | `88`                      |

追加が成功すると一覧が自動で再取得され、新しい行が追加されます。

> **バリデーションエラーが出た場合**  
> 同じメールアドレスはすでに登録されています。別のアドレスを使ってください。

---

#### ④ 生徒を削除する

削除したい行の「削除」ボタンを押します。  
確認ダイアログが表示されるので「OK」を押すと削除され、一覧が再取得されます。

---

#### ⑤ ログアウトしてトークンが無効になることを確認する

1. ヘッダー右側の「ログアウト（POST /api/logout）」ボタンを押す
2. ログイン画面に戻れば成功

> **なぜ一覧が見えなくなる？**  
> `POST /api/logout` を呼ぶと、サーバーの `personal_access_tokens` テーブルから  
> そのトークンのレコードが削除されます。  
> 削除されたトークンはどこに送っても認証されなくなるため、一覧も取得できなくなります。

### fetch() とは（React への布石）

このページは JavaScript の `fetch()` を使って API を呼んでいます。  
ページ下部の「▶ このページが使っているコード」をクリックすると実際のコードが確認できます。

```javascript
// Authorization ヘッダーにトークンを付けてリクエストを送る
const res = await fetch('/api/students', {
    headers: {
        'Accept':        'application/json',
        'Authorization': `Bearer ${token}`,  // ← このトークンをヘッダーに付ける
    },
});
const data = await res.json();
// data.data に生徒の配列が入っている
```

> React の学習（コマ17〜）でこのコードが登場したとき、  
> 「このページで動いていたやつだ」と気づけるはずです。
