## コマ17：Mock版を API に繋ぎ替える

### 今日のゴール
- Next.js プロジェクトを作成できる
- App Router の仕組みを理解している
- コマ16 で作った Mock 版の画面を、本物の Laravel API に繋ぎ替えられる

> **今日のポイント：**  
> コマ16で作ったコンポーネント（LoginForm・StudentList・AddStudentForm）は  
> **ほぼそのまま使えます。** 変わるのはデータの取り方だけです。

---

### 0. Mock版とAPI版の違いを確認する

| | コマ16（Mock版・React） | コマ17（API版・Next.js） |
|--|--|--|
| データの取得 | コード内の固定配列 | `fetch()` で Laravel API から取得 |
| ログイン | 固定値と照合 | `POST /api/login` でトークン取得 |
| 生徒の追加 | 配列に追加 | `POST /api/students` を呼ぶ |
| 生徒の削除 | 配列から除外 | `DELETE /api/students/{id}` を呼ぶ |
| 画面の見た目 | **全く同じ** | **全く同じ** |

### 1. Next.js プロジェクトを作成する

#### 1-1. インストール

ターミナルで以下を実行します（`my-app` とは **別のフォルダ** に作ります）。

```bash
npx create-next-app@latest school-frontend
```

いくつか質問が出る時があります。質問が出た際は以下のように答えてください。

```
Would you like to use TypeScript?             → Yes  ← TypeScript を使う
Would you like to use ESLint?                 → Yes
Would you like to use Tailwind CSS?           → Yes  ← インストールしておく
Would you like your code inside a src/ dir?   → No   ← ルート直下に app/ フォルダを置く
Would you like to use App Router?             → Yes
Would you like to use Turbopack?              → No
Would you like to customize the import alias? → No
```

> **TypeScript について**  
> ファイルの拡張子が `.tsx`（TypeScript）になります。基本的なコードは JavaScript とほぼ同じです。  
> 型の記述が必要な箇所は、その都度説明します。
>
> **Tailwind CSS について**  
> インストールはしますが、今回はインラインスタイルを使います。Tailwind は今後の学習で活用できます。

#### 1-2. インストールして起動する

```bash
# プロジェクトフォルダに移動
cd school-frontend

# 開発サーバーを起動（create-next-app が自動でパッケージをインストール済み）
npm run dev
```

`http://localhost:3000` を開いて Next.js のサンプル画面が表示されれば成功です。

### 3. コマ16のコンポーネントをコピーする

まず `src/components/` フォルダを作り、コマ16で作ったファイルをコピーします。

```
school-frontend/components/ に以下をコピー：
  ├── LoginForm.tsx      ← コマ16の LoginForm.jsx をコピー（拡張子を .tsx に変更）
  ├── StudentList.tsx    ← コマ16の StudentList.jsx をコピー（拡張子を .tsx に変更）
  └── AddStudentForm.tsx ← コマ16の AddStudentForm.jsx をコピー（拡張子を .tsx に変更）
```

> **拡張子について**  
> TypeScript を使うため、ファイル拡張子は `.tsx`（JSX を含むファイル）に統一します。  
> コード内容はコマ16の `.jsx` からほぼそのままコピーできます。

コピーしたファイルそれぞれの **先頭1行目** に `"use client";` を追加します。

```js
"use client";  // ← この1行を追加！
import { useState } from "react";
// ... 以下はコマ16のコードそのまま
```

> **`"use client"` とは？**  
> Next.js はデフォルトでサーバー側でコードを動かします。  
> `useState` や `onClick` など「ブラウザの機能」を使うコンポーネントには  
> `"use client"` を先頭に書いて「これはブラウザで動かす」と伝えます。

### 6. ログインページを作る（/login）

`app/login/` フォルダを作成し、`page.tsx` を新規作成します。

```
app/login/page.tsx  ← 新規作成
```

以下をコピーして貼り付けます。

```tsx
"use client";

import { useState } from "react";
import { useRouter } from "next/navigation";

export default function LoginPage() {
  const router = useRouter(); // ページ移動に使う
  const [email, setEmail] = useState("test@example.com");
  const [password, setPassword] = useState("password");
  const [error, setError] = useState("");
  const [loading, setLoading] = useState(false);

  async function handleSubmit(e) {
    e.preventDefault();
    setError("");
    setLoading(true);

    // ★ コマ16との違い：固定値チェック → 本物のAPIを呼ぶ
    const res = await fetch("http://localhost:8000/api/login", {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        "Accept": "application/json",
      },
      body: JSON.stringify({ email, password }),
    });

    const data = await res.json();
    setLoading(false);

    if (res.ok) {
      // ★ トークンをブラウザに保存しておく
      localStorage.setItem("token", data.token);
      localStorage.setItem("userName", data.user?.name || "ユーザー");
      // /students ページに移動
      router.push("/students");
    } else {
      setError("メールアドレスまたはパスワードが違います");
    }
  }

  return (
    <div style={{
      maxWidth: "400px", margin: "80px auto",
      background: "white", padding: "32px",
      borderRadius: "8px", boxShadow: "0 2px 8px rgba(0,0,0,0.1)"
    }}>
      <h2 style={{ marginBottom: "24px", textAlign: "center" }}>ログイン</h2>

      <form onSubmit={handleSubmit}>
        <div style={{ marginBottom: "16px" }}>
          <label style={{ display: "block", marginBottom: "6px", fontWeight: "bold" }}>
            メールアドレス
          </label>
          <input
            type="email"
            value={email}
            onChange={(e) => setEmail(e.target.value)}
            style={{ width: "100%", padding: "10px", border: "1px solid #ccc", borderRadius: "6px", fontSize: "1em", boxSizing: "border-box" }}
          />
        </div>

        <div style={{ marginBottom: "16px" }}>
          <label style={{ display: "block", marginBottom: "6px", fontWeight: "bold" }}>
            パスワード
          </label>
          <input
            type="password"
            value={password}
            onChange={(e) => setPassword(e.target.value)}
            style={{ width: "100%", padding: "10px", border: "1px solid #ccc", borderRadius: "6px", fontSize: "1em", boxSizing: "border-box" }}
          />
        </div>

        {error && (
          <p style={{ color: "#D32F2F", fontSize: "0.9em", marginBottom: "12px" }}>
            {error}
          </p>
        )}

        <button
          type="submit"
          disabled={loading}
          style={{ width: "100%", padding: "12px", background: loading ? "#aaa" : "#1976D2", color: "white", border: "none", borderRadius: "6px", fontSize: "1em", fontWeight: "bold", cursor: loading ? "not-allowed" : "pointer" }}
        >
          {loading ? "ログイン中..." : "ログイン"}
        </button>
      </form>
    </div>
  );
}
```

**コマ16の `LoginForm.tsx` との違い：**

| | コマ16（Mock） | コマ17（API） |
|--|--|--|
| 認証方法 | 固定値と照合 | `fetch("http://localhost:8000/api/login")` |
| ログイン後 | `setIsLoggedIn(true)` | `localStorage.setItem("token", ...)` + ページ移動 |
| ページ移動 | 条件分岐で画面切り替え | `router.push("/students")` |

### 7. 生徒管理ページを作る（/students）

`app/students/` フォルダを作成し、`page.tsx` を新規作成します。

```
app/students/page.tsx  ← 新規作成
```

以下をコピーして貼り付けます。

```tsx
"use client";

import { useState, useEffect } from "react";
import { useRouter } from "next/navigation";
import StudentList from "@/components/StudentList";
import AddStudentForm from "@/components/AddStudentForm";

export default function StudentsPage() {
  const router = useRouter();
  const [students, setStudents] = useState([]);
  const [userName, setUserName] = useState("");
  const [loading, setLoading] = useState(true);
  const [addError, setAddError] = useState(""); // ← 追加エラー表示用

  // ★ useEffect：ページが表示されたとき1回だけ実行される
  useEffect(() => {
    const token = localStorage.getItem("token");

    // トークンがなければログインページに戻す
    if (!token) {
      router.push("/login");
      return;
    }

    setUserName(localStorage.getItem("userName") || "ユーザー");
    loadStudents(token);
  }, []); // [] = 最初の1回だけ実行

  // 生徒一覧をAPIから取得する
  async function loadStudents(token) {
    setLoading(true);

    const res = await fetch("http://localhost:8000/api/students", {
      headers: {
        "Accept": "application/json",
        "Authorization": `Bearer ${token}`, // ★ トークンをヘッダーに付ける
      },
    });

    // 401 = トークンが無効 → ログインページに戻す
    if (res.status === 401) {
      router.push("/login");
      return;
    }

    const data = await res.json();
    setStudents(data.data); // Laravel API は { data: [...] } の形で返す
    setLoading(false);
  }

  // 生徒を追加する
  async function handleAdd(newStudent) {
    const token = localStorage.getItem("token");
    setAddError(""); // ← エラーをリセット

    const res = await fetch("http://localhost:8000/api/students", {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        "Accept": "application/json",
        "Authorization": `Bearer ${token}`,
      },
      body: JSON.stringify(newStudent),
    });

    if (res.ok) {
      loadStudents(token); // ★ 追加後に一覧を再取得して画面を更新
    } else {
      // ★ 失敗したときはAPIのエラー内容を画面に表示する
      const data = await res.json();
      if (data.errors) {
        const messages = Object.values(data.errors).flat().join(" / ");
        setAddError(messages);
      } else {
        setAddError(`エラーが発生しました（HTTP ${res.status}）`);
      }
    }
  }

  // 生徒を削除する
  async function handleDelete(id) {
    const token = localStorage.getItem("token");

    const res = await fetch(`http://localhost:8000/api/students/${id}`, {
      method: "DELETE",
      headers: {
        "Accept": "application/json",
        "Authorization": `Bearer ${token}`,
      },
    });

    if (res.ok) {
      loadStudents(token); // ★ 削除後に一覧を再取得して画面を更新
    }
  }

  // ログアウト
  async function handleLogout() {
    const token = localStorage.getItem("token");

    await fetch("http://localhost:8000/api/logout", {
      method: "POST",
      headers: {
        "Accept": "application/json",
        "Authorization": `Bearer ${token}`,
      },
    });

    localStorage.removeItem("token");
    localStorage.removeItem("userName");
    router.push("/login");
  }

  if (loading) {
    return (
      <div style={{ textAlign: "center", padding: "80px", color: "#888" }}>
        読み込み中...
      </div>
    );
  }

  return (
    <div style={{ background: "#f4f6f8", minHeight: "100vh" }}>

      {/* ヘッダー（コマ16と同じ見た目） */}
      <header style={{
        background: "#1565C0", color: "white",
        padding: "16px 32px",
        display: "flex", alignItems: "center"
      }}>
        <h1 style={{ fontSize: "1.2em" }}>生徒管理アプリ（Next.js API版）</h1>
        <div style={{ marginLeft: "auto", display: "flex", alignItems: "center", gap: "12px" }}>
          <span style={{ fontSize: "0.9em" }}>● ログイン中：{userName}</span>
          <button
            onClick={handleLogout}
            style={{ background: "#D32F2F", color: "white", border: "none", borderRadius: "6px", padding: "6px 14px", cursor: "pointer" }}
          >
            ログアウト
          </button>
        </div>
      </header>

      {/* メインコンテンツ（コマ16と同じコンポーネントを使う） */}
      <main style={{ padding: "32px", maxWidth: "900px", margin: "0 auto" }}>
        <AddStudentForm onAdd={handleAdd} />
        {/* ★ 追加失敗時にAPIのエラーメッセージを表示する */}
        {addError && (
          <p style={{ color: "#D32F2F", marginBottom: "16px", fontWeight: "bold" }}>
            ⚠ {addError}
          </p>
        )}
        <StudentList students={students} onDelete={handleDelete} />
      </main>

    </div>
  );
}
```

**コマ16の `App.tsx` との違い：**

|                            | コマ16（Mock）            | コマ17（API）                         |
| -------------------------- | --------------------- | --------------------------------- |
| 初期データ                      | `INITIAL_STUDENTS` 配列 | `useEffect` → `fetch` で取得         |
| 追加                         | 配列に push              | `POST /api/students` → 再取得        |
| 削除                         | 配列を filter            | `DELETE /api/students/{id}` → 再取得 |
| StudentList/AddStudentForm | **全く同じ**              | **全く同じ**                          |

### 8. CORS 設定（Laravel 側）

Next.js（`http://localhost:3000`）から Laravel（`http://localhost:8000`）を呼ぶとき、  
**ブラウザのセキュリティ機能（CORS）** により、通常は通信がブロックされます。

**Laravel 13 ではデフォルトで全オリジン（`*`）が許可されているため、開発中は設定変更不要です。**

> **CORS とは？**  
> ブラウザが「別のサーバー（ポートが違うだけでも別扱い）へのリクエスト」を  
> セキュリティのためにブロックする仕組みです。  
> Laravel 側で「このオリジンからのアクセスは OK」と明示することで解除できます。

---

#### （参考）本番環境では接続元を限定する

開発が終わって本番公開するときは、アクセスを許可するオリジンを絞り込みます。

まず設定ファイルを生成します：

```bash
php artisan config:publish cors
```

生成された `config/cors.php` を開いて変更します：

```php
// 変更前（全オリジン許可）
'allowed_origins' => ['*'],

// 変更後（本番フロントエンドの URL のみ許可）
'allowed_origins' => ['https://your-frontend.com'],
```

変更後にキャッシュをクリアします：

```bash
php artisan config:clear
```

### 8-2. ルート設定のよくあるミス（`routes/api.php`）

> **⚠ ハマりポイント：`Route::apiResource` の二重登録**  
>
> 以下のように `students` ルートを2回書くと、**後から書いた方が優先**されます。  
> 認証なし版と認証あり版が混在すると、意図しない動作になります。
>
> ```php
> // ❌ NG：同じリソースが2回登録されている
> Route::apiResource('students', StudentController::class); // ← 削除する
>
> Route::middleware('auth:sanctum')->group(function () {
>     Route::apiResource('students', StudentController::class); // ← こちらだけ残す
> });
> ```
>
> ```php
> // ✅ OK：auth:sanctum の中だけに書く
> Route::post('/login', [AuthController::class, 'login']);
>
> Route::middleware('auth:sanctum')->group(function () {
>     Route::post('/logout', [AuthController::class, 'logout']);
>     Route::get('/me',      [AuthController::class, 'me']);
>     Route::apiResource('students', StudentController::class);
> });
> ```
>
> 設定後は `php artisan route:list --path=api/students` でルート一覧を確認できます。

### Button コンポーネントの例

```tsx
"use client";

// variant = "primary"（青）または "danger"（赤）
function Button({ children, onClick, variant = "primary", disabled = false }: {
  children: React.ReactNode;
  onClick?: () => void;
  variant?: "primary" | "danger";
  disabled?: boolean;
}) {
  const bgColor = variant === "danger" ? "#D32F2F" : "#1976D2";

  return (
    <button
      onClick={onClick}
      disabled={disabled}
      style={{
        padding: "9px 18px",
        background: disabled ? "#aaa" : bgColor,
        color: "white",
        border: "none",
        borderRadius: "6px",
        fontWeight: "bold",
        cursor: disabled ? "not-allowed" : "pointer",
```

## コマ19：useEffect の深掘り

### 今日のゴール
- `useEffect` の動くタイミングを説明できる
- 依存配列（第2引数）の意味を理解できる

---

### useEffect とは

```js
useEffect(() => {
  // ここに書いた処理が「特定のタイミング」で実行される
}, [依存配列]);
```

| 依存配列 | 実行タイミング |
|---------|-------------|
| `[]` | コンポーネントが最初に表示されたとき **1回だけ** |
| `[value]` | `value` が変化するたびに実行 |
| 省略（なし） | 毎レンダリング後に実行（ほぼ使わない） |

---

### よくある使い方：IDが変わったら再取得

```tsx
"use client";

import { useState, useEffect } from "react";

export default function StudentDetailPage({ params }: { params: { id: string } }) {
  const [student, setStudent] = useState<any>(null);

  useEffect(() => {
    const token = localStorage.getItem("token");

    fetch(`http://localhost:8000/api/students/${params.id}`, {
      headers: {
        "Accept": "application/json",
        "Authorization": `Bearer ${token}`,
      },
    })
      .then((res) => res.json())
      .then((data) => setStudent(data.data));

  }, [params.id]); // ← params.id が変わるたびに再取得

  if (!student) return <p>読み込み中...</p>;

  return (
    <div>
      <h1>{student.name}</h1>
      <p>メール: {student.email}</p>
      <p>点数: {student.score}</p>
    </div>
  );
}
```

## コマ20：認証の仕組みを整理する

### 今日のゴール
- トークン認証の流れを図で説明できる
- 未ログイン時のリダイレクトを実装できる

---

### トークン認証の流れ

```
① ログイン
   Next.js  →  POST /api/login  →  Laravel
   Next.js  ←  { token: "xxx" } ←  Laravel
   Next.js: localStorage.setItem("token", "xxx")

② データ取得
   Next.js  →  GET /api/students
               Authorization: Bearer xxx
            →  Laravel（トークン検証）
   Next.js  ←  { data: [...] }  ←  Laravel

③ ログアウト
   Next.js  →  POST /api/logout
               Authorization: Bearer xxx
            →  Laravel（トークン削除）
   Next.js: localStorage.removeItem("token")
```

---

### 未ログイン時のリダイレクト（カスタムフック）

同じリダイレクト処理を複数ページで使うなら、**カスタムフック** にまとめます。

```ts
// hooks/useAuth.ts
import { useEffect, useState } from "react";
import { useRouter } from "next/navigation";

export function useAuth() {
  const router = useRouter();
  const [token, setToken] = useState(null);
  const [userName, setUserName] = useState("");

  useEffect(() => {
    const savedToken = localStorage.getItem("token");
    if (!savedToken) {
      router.push("/login");
      return;
    }
    setToken(savedToken);
    setUserName(localStorage.getItem("userName") || "ユーザー");
  }, []);

  return { token, userName };
}

// 使い方（各ページで）
// const { token, userName } = useAuth();
// → トークンがなければ自動で /login に飛ぶ
```

### バリデーションエラーを Laravel から受け取る

Laravel は 422 エラーのとき、レスポンスボディに失敗した項目と理由を返します。

```json
{
  "message": "The email has already been taken.",
  "errors": {
    "email": ["The email has already been taken."]
  }
}
```

これを画面に表示するには、`data.errors` をループして取り出します。

```tsx
if (res.ok) {
  loadStudents(token);
} else {
  const data = await res.json();
  if (data.errors) {
    // errors の値（配列）をすべて結合して1つの文字列にする
    const messages = Object.values(data.errors).flat().join(" / ");
    setAddError(messages);
  } else {
    setAddError(`エラーが発生しました（HTTP ${res.status}）`);
  }
}
```

> **⚠ よくある 422 の原因：メールアドレスの重複**  
> Laravel の `unique:students` ルールにより、同じメールアドレスは1件しか登録できません。  
> 「The email has already been taken.」と表示された場合は、別のメールアドレスを使ってください。
>
> | エラーメッセージ | 原因 |
> |--|--|
> | The email has already been taken. | そのメールアドレスは既にDBに存在する |
> | The score must be an integer. | 点数に小数（例：80.5）が入っている |
> | The name field is required. | 名前が空欄 |
