### ブランチとは

**ブランチ = 作業の「分岐」。メインのコードを壊さずに新機能を開発できる**

```
main（本番ブランチ）
  │
  ├── feature/login    ← ログイン機能を開発中
  ├── feature/search   ← 検索機能を開発中
  └── bugfix/404-error ← バグ修正中
```

```bash
# 現在のブランチを確認
git branch
# → * main（* が現在のブランチ）

# 新しいブランチを作成して移動
git checkout -b feature/add-profile
# または（Git 2.23以降）
git switch -c feature/add-profile

# ブランチを一覧表示
git branch -a   # -a でリモートブランチも表示

# main に戻る
git switch main

# ブランチを削除
git branch -d feature/add-profile
```

---

### merge — ブランチを結合する

```bash
# ① feature/add-profile ブランチで作業してコミット
git switch feature/add-profile
# ... コードを書く ...
git add .
git commit -m "プロフィール画面を追加"

# ② main ブランチに戻る
git switch main

# ③ feature/add-profile の変更を main に取り込む
git merge feature/add-profile

# ④ マージ後にブランチを削除（任意）
git branch -d feature/add-profile
```

---

### コンフリクト（衝突）の解決

```bash
# 2人が同じ行を変更した場合、マージ時にコンフリクトが起きる
# コンフリクトしたファイルを開くと...

<<<<<<< HEAD
  echo "Aさんが書いたコード";
=======
  echo "Bさんが書いたコード";
>>>>>>> feature/b-work

# ↑ これを手動で正しいコードに直す
# 直したら：
git add .
git commit -m "コンフリクトを解消"
```

## コマ02：GitHub Flow — PR・コードレビュー

### GitHub Flow とは

チームで開発するための決まりごと（ルール）です。

```
① main から作業ブランチを切る
      git switch -c feature/xxxx

② ブランチで作業・コミット
      git add .
      git commit -m "メッセージ"

③ GitHub にプッシュ
      git push origin feature/xxxx

④ Pull Request（PR）を作成
      → GitHub 上で「このコードを main に入れたい」と申請する

⑤ チームメンバーがコードレビュー
      → コメントを書いたり、修正を依頼したりする

⑥ 問題なければ main にマージ
⑦ デプロイ（本番に反映）
```

### Pull Request の書き方

```markdown
## 変更内容
- プロフィール画面を追加
- ユーザーの名前・アイコンを表示できるようにした

## 確認してほしい点
- `/profile` にアクセスしたとき正しく表示されるか
- 未ログイン時は `/login` にリダイレクトされるか

## スクリーンショット
（画像を添付）
```

### コードレビューのポイント

- バグや論理エラーがないか
- セキュリティの問題がないか（XSS、SQL Injection など）
- コードは読みやすいか
- テストは書かれているか（2年生では任意）

## コマ03：1年生アプリをチームで動かす

### 手順

```bash
# ① チームリーダーが1年生のアプリを GitHub に push する
git init
git remote add origin https://github.com/チーム名/app.git
git add .
git commit -m "initial commit"
git push -u origin main

# ② チームメンバーが clone する
git clone https://github.com/チーム名/app.git
cd app

# ③ 依存関係のインストール
composer install
npm install

# ④ 環境設定
cp .env.example .env
php artisan key:generate

# ⑤ DBの設定（.env を編集）
# DB_DATABASE=app_db
# DB_USERNAME=root
# DB_PASSWORD=

# ⑥ マイグレーション
php artisan migrate

# ⑦ 起動確認
php artisan serve
```

## まとめ

| コマンド | 意味 |
|---------|------|
| `git switch -c ブランチ名` | ブランチを作成して移動 |
| `git switch main` | main に戻る |
| `git merge ブランチ名` | ブランチを現在のブランチに結合 |
| `git push origin ブランチ名` | リモートにブランチを push |
| Pull Request（PR） | GitHub でコードのマージを申請する |
| コードレビュー | PR に対してコメント・指摘を行う |
