
## BladeアプリをAPI化

### 今日のゴール
- REST API とは何かを説明できる
- JSON レスポンスを返すコントローラが書ける
- `api.php` のルーティングを理解している

---

### REST API とは

#### 「API って何？」をひとことで言うと

**API = 「データだけ返す窓口」**

1年生では、ブラウザで URL にアクセスすると HTML（見た目込みのページ）が返ってきました。  
API は同じ仕組みを使いながら、HTML の代わりに**データ（JSON）だけ**を返します。

```
【Blade アプリ（1年生で作ったもの）】
ブラウザ → GET /students → サーバー → HTML ページ を返す
                                      ↑ 表・ボタン・デザインなど全部入り

【REST API（今日から作る）】
ブラウザや他のアプリ → GET /api/students → サーバー → JSON データ だけを返す
                                                      ↑ 見た目なし・データのみ
```

> **なぜ JSON だけ返すの？**  
> React や Vue などのフロントエンドは「自分でデザインを作る」ので、  
> サーバーには「データだけ」送ってほしいのです。  
> API を作ると「Reactで動くアプリ」「スマホアプリ」など様々な画面から同じデータを使い回せます。

---

#### JSON ってどんな形？

**JSON（ジェイソン）** は、データを表すテキスト形式です。  
ブラウザや他のアプリが読みやすいように、決まった書き方があります。

```json
{
    "status": "ok",
    "data": [
        {
            "id": 1,
            "name": "田中太郎",
            "email": "tanaka@example.com",
            "score": 85
        },
        {
            "id": 2,
            "name": "山田花子",
            "email": "yamada@example.com",
            "score": 72
        }
    ]
}
```

> **見方のポイント：**  
> - `{ }` の中に「キー」: 「値」の形で書く  
> - `[ ]` はリスト（複数のデータをまとめる）  
> - 文字列は `"` で囲む。数字は囲まない  
> - 最後の項目の後にカンマ `,` は付けない

---

#### REST の考え方：URL と HTTP メソッドの組み合わせ

REST API は「**URL（どこに）** × **HTTP メソッド（何をする）**」でやることを決めます。  
HTTP メソッドとは、リクエストの種類を表す言葉です。

| HTTP メソッド | 意味 | 日常的な操作に例えると |
|-------------|------|-------------------|
| `GET`       | 取得する | ページを見る・検索する |
| `POST`      | 新規作成する | フォームを送信する |
| `PUT/PATCH` | 更新する | 内容を書き換える |
| `DELETE`    | 削除する | 削除ボタンを押す |

この授業で作る生徒 API の一覧：

| URL                  | メソッド      | 意味   | 何をするか           |
| -------------------- | --------- | ---- | --------------- |
| `/api/students`      | GET       | 一覧取得 | 全生徒のデータを返す      |
| `/api/students`      | POST      | 新規作成 | 新しい生徒を登録する      |
| `/api/students/{id}` | GET       | 詳細取得 | 指定した1人の生徒を返す    |
| `/api/students/{id}` | PUT/PATCH | 更新   | 指定した生徒の情報を書き換える |
| `/api/students/{id}` | DELETE    | 削除   | 指定した生徒を削除する     |

> `{id}` は数字が入ります。例えば `GET /api/students/3` は「id が 3 の生徒を取得する」という意味です。

---

### HTTP ステータスコードについて

API のレスポンスには必ず**ステータスコード**（数字）が付きます。  
ブラウザや React がこの数字を見て「成功か失敗か」を判断します。

| コード | 意味 | いつ使う |
|-------|------|---------|
| `200 OK` | 成功 | 取得・更新・削除の成功 |
| `201 Created` | 作成成功 | POST で新規作成したとき |
| `404 Not Found` | 見つからない | 指定した id が存在しないとき |
| `422 Unprocessable Entity` | バリデーションエラー | 入力値が正しくないとき |
| `500 Internal Server Error` | サーバーエラー | プログラムにバグがあるとき |

> 例えば `GET /api/students/999` で id=999 の生徒が存在しない場合、  
> 「404」を返すことで「見つかりませんでした」を伝えます。

### Step 10｜API コントローラを作成する

#### 10-1. コントローラファイルを作る

ターミナルで以下を実行します（`php artisan serve` を実行中のターミナルとは**別のタブ**を開いてください）。

```bash
php artisan make:controller Api/StudentController --api
```

**コマンドの意味：**
- `make:controller` → コントローラファイルを作る
- `Api/StudentController` → `Api` フォルダの中に `StudentController.php` を作る
- `--api` → API 用の5つのメソッドをあらかじめ書いてくれる（`index` `store` `show` `update` `destroy`）

実行後に以下が表示されれば成功です：

```
   INFO  Controller [app/Http/Controllers/Api/StudentController.php] created successfully.
```

作成されたファイルの場所を確認します：

```
（リポジトリ名）/
└── app/
    └── Http/
        └── Controllers/
            └── Api/                     ← 新しく作られたフォルダ
                └── StudentController.php  ← これが作られた
```

---

#### 10-2. コントローラファイルを開く

VSCode で `app/Http/Controllers/Api/StudentController.php` を開きます。  
`--api` オプションで作られたファイルは、最初からこのような内容になっています：

```php
<?php

namespace App\Http\Controllers\Api;

use Illuminate\Http\Request;
use App\Http\Controllers\Controller;

class StudentController extends Controller
{
    public function index()
    {
        //
    }

    public function store(Request $request)
    {
        //
    }

    public function show(string $id)
    {
        //
    }

    public function update(Request $request, string $id)
    {
        //
    }

    public function destroy(string $id)
    {
        //
    }
}
```

> **`namespace` と `use` って何？**  
> - `namespace` はこのファイルの「住所」のようなものです。`Api` フォルダの中にいることを表しています。  
> - `use` は他のファイルを「呼び出す宣言」です。`Student` モデルを使うために後で追加します。

---

#### 10-3. コントローラに処理を書く

ファイルの内容を**全て**以下のコードに書き替えてください。  
（Ctrl+A で全選択 → 貼り付け、または既存の内容を消して書き替えます）

```php
<?php

namespace App\Http\Controllers\Api;

use App\Http\Controllers\Controller;
use App\Models\Student;           // ← Student モデルを使えるようにする
use Illuminate\Http\Request;

class StudentController extends Controller
{
    // -------------------------------------------------------
    // GET /api/students  → 全生徒の一覧を返す
    // -------------------------------------------------------
    public function index()
    {
        $students = Student::all();  // データベースから全件取得

        return response()->json([
            'status' => 'ok',
            'data'   => $students,   // 取得したデータを JSON に入れて返す
        ]);
    }

    // -------------------------------------------------------
    // POST /api/students  → 新しい生徒を登録する
    // -------------------------------------------------------
    public function store(Request $request)
    {
        // バリデーション：送られてきたデータの形式を確認する
        $request->validate([
            'name'  => 'required|max:100',          // 必須・100文字以内
            'email' => 'required|email|unique:students', // 必須・メール形式・重複不可
            'score' => 'required|integer|min:0|max:100', // 必須・整数・0〜100
        ]);

        $student = Student::create($request->all()); // DB に保存

        // 201 Created（作成成功）のステータスコードで返す
        return response()->json([
            'status' => 'ok',
            'data'   => $student,
        ], 201);
    }

    // -------------------------------------------------------
    // GET /api/students/{id}  → id で指定した1人を返す
    // -------------------------------------------------------
    public function show($id)
    {
        $student = Student::find($id); // id で1件検索

        if (!$student) {
            // 見つからなかった場合 → 404 Not Found を返す
            return response()->json([
                'status'  => 'error',
                'message' => 'Student not found',
            ], 404);
        }

        return response()->json([
            'status' => 'ok',
            'data'   => $student,
        ]);
    }

    // -------------------------------------------------------
    // PUT /api/students/{id}  → id で指定した生徒を更新する
    // -------------------------------------------------------
    public function update(Request $request, $id)
    {
        $student = Student::findOrFail($id); // 見つからなければ自動で 404 を返す

        $request->validate([
            'name'  => 'sometimes|max:100',          // sometimes = 送られてきた場合だけチェック
            'score' => 'sometimes|integer|min:0|max:100',
        ]);

        $student->update($request->all()); // DB を更新

        return response()->json([
            'status' => 'ok',
            'data'   => $student,
        ]);
    }

    // -------------------------------------------------------
    // DELETE /api/students/{id}  → id で指定した生徒を削除する
    // -------------------------------------------------------
    public function destroy($id)
    {
        Student::findOrFail($id)->delete(); // 見つけて削除

        return response()->json([
            'status'  => 'ok',
            'message' => 'Deleted',
        ]);
    }
}
```

**書き替えたら Ctrl+S（Mac は Cmd+S）で保存します。**

---

#### 各メソッドの役割まとめ

| メソッド名 | 対応する URL | やること |
|-----------|------------|---------|
| `index()` | GET /api/students | 全件取得して JSON で返す |
| `store()` | POST /api/students | バリデーション → DB保存 → 201で返す |
| `show($id)` | GET /api/students/{id} | 1件検索 → なければ404 → JSONで返す |
| `update($request, $id)` | PUT /api/students/{id} | 1件検索 → バリデーション → DB更新 |
| `destroy($id)` | DELETE /api/students/{id} | 1件検索 → 削除 |

### Step 11｜API ルーティングを有効にする

コントローラを作っただけでは、URL とコントローラがつながっていません。  
「この URL にアクセスしたらこのメソッドを呼ぶ」という**ルーティング**を設定します。

#### 11-1. `routes/api.php` を作成する

> **⚠️ Laravel 11 以降、`routes/api.php` は最初から存在しません。**  
> 以下のコマンドを実行して初めて作成されます。

```bash
php artisan install:api
```

実行すると、途中で2回確認を求められます。どちらも `yes`（または Enter キー）で進んでください。

---

**【確認1】マイグレーションを実行するか？**

```
   INFO  Published API routes file.

 One new database migration has been published. Would you like to run all pending database migrations? (yes/no) [yes]:
 > yes
```

> `routes/api.php` が作成されたあと、マイグレーションを今すぐ実行するか聞かれます。  
> `yes` または Enter で進みます。

---

**【確認2】本番環境での実行確認**

```
                                                                    
                         APPLICATION IN PRODUCTION.                 
                                                                    

 ┌ Are you sure you want to run this command? ──────────────────┐
 │ Yes                                                          │
 └──────────────────────────────────────────────────────────────┘
```

> **「APPLICATION IN PRODUCTION（本番環境です）」と表示されて驚くかもしれませんが、心配不要です。**  
> これは「本当に実行していいですか？」という Laravel の安全確認です。  
> 「Yes」を選んで進めてください。

---

**マイグレーションが実行される**

```
   INFO  Running migrations.

  2026_xx_xx_xxxxxx_create_personal_access_tokens_table .......... DONE
```

> `personal_access_tokens`（パーソナルアクセストークン）テーブルが作られます。  
> これは **Sanctum が認証用のトークンを保存するためのテーブル**です。  
> コマ07の認証機能を実装するときに使います。今は「自動で作られた」とだけ覚えておけば大丈夫です。

---

**最後にこのメッセージが表示されれば成功：**

```
   INFO  API scaffolding installed. Please add the [Laravel\Sanctum\HasApiTokens] trait to your User model.
```

> 末尾の `Please add the [Laravel\Sanctum\HasApiTokens] trait...` は  
> 「User モデルに設定を追加してください」という案内ですが、**コマ07で行います。今は無視して大丈夫です。**

---

このコマンドが行ったこと：
- `routes/api.php` ファイルの作成
- `bootstrap/app.php` に API ルーティングを登録
- Laravel Sanctum（コマ07で使う認証パッケージ）のインストール
- `personal_access_tokens` テーブルのマイグレーション実行

実行後のフォルダ構成：

```
（リポジトリ名）/
└── routes/
    ├── api.php   ← これが作成された（API のルーティング）
    └── web.php   ← こちらは Blade 用（今回は触らない）
```

> **`api.php` と `web.php` の違い**  
> - `web.php` … HTML を返すルートを書く  
> - `api.php` … JSON を返す API 用。`/api/` が URL の先頭に自動で付く

#### 11-2. ルートを追加する

VSCode で `routes/api.php` を開きます。最初はほぼ空です。以下のように書き替えます：

```php
<?php

use Illuminate\Support\Facades\Route;
use App\Http\Controllers\Api\StudentController;  // ← この行を追加

Route::apiResource('students', StudentController::class);
// ↑ この1行だけで5つのルートが自動で登録される
```

**書き替えたら Ctrl+S（Mac は Cmd+S）で保存します。**

---

#### `Route::apiResource` が登録するルートを確認する

`Route::apiResource('students', StudentController::class)` は、以下の5つのルートを**1行で**まとめて登録します。

確認コマンド：

```bash
php artisan route:list --path=api/students
```

```
  GET|HEAD   api/students ............. students.index  › Api\StudentController@index
  POST       api/students ............. students.store  › Api\StudentController@store
  GET|HEAD   api/students/{student} ... students.show   › Api\StudentController@show
  PUT|PATCH  api/students/{student} ... students.update › Api\StudentController@update
  DELETE     api/students/{student} ... students.destroy› Api\StudentController@destroy
```

> これを1行ずつ手で書くと5行になりますが、`apiResource` を使うと1行で済みます。

開発サーバーが起動していない場合は、起動します：


#### 12-1. 一覧取得（GET /api/students）をブラウザで確認する

ブラウザで以下の URL にアクセスします：

```
http://localhost:8000/api/students
```

以下のような JSON が表示されれば成功です：

```json
{
    "status": "ok",
    "data": [
        {
            "id": 1,
            "name": "Tanaka Taro",
            "email": "tanaka@example.com",
            "score": 72,
            "created_at": "2024-xx-xx ...",
            "updated_at": "2024-xx-xx ..."
        },
        ...
    ]
}
```

> **JSON が表示されずにエラーが出た場合は以下を確認してください。**

**よくあるエラーと対処法：**

| エラー内容 | 原因 | 対処法 |
|-----------|------|-------|
| `Class "App\Http\Controllers\Api\StudentController" not found` | use の書き方が間違っている | `api.php` の `use` の行を確認する |
| `SQLSTATE[HY000] [2002] Connection refused` | MySQL が起動していない | MySQL を起動する（XAMPP or `brew services start mysql`） |
| 空白ページ・500エラー | PHP の構文エラー | `.storage/logs/laravel.log` を開いてエラーを確認する |
| `{"message":""}` のみ表示 | ルートは見つかったが処理でエラー | コントローラの内容を確認する |

#### 12-3. POST・PUT・DELETE は curl コマンドで確認する

ブラウザのアドレスバーは GET リクエストしか送れません。  
POST（新規作成）・PUT（更新）・DELETE（削除）は **curl（カール）** というコマンドを使って確認します。

> **curl とは？**  
> ターミナルから HTTP リクエストを送るコマンドです。Mac・Windows どちらにも標準で入っています。  
> `php artisan serve` を実行中のターミナルとは**別のタブ**で実行してください。

---

**curl コマンドのオプションの読み方：**

| オプション | 意味 |
|-----------|------|
| `-X POST` | HTTP メソッドを指定する（POST / PUT / DELETE など） |
| `-H "Content-Type: application/json"` | 「JSON を送ります」という宣言 |
| `-d '{...}'` | 送るデータ（JSON 形式） |

---

**① 新規作成（POST /api/students）**

**Mac の場合：**
```bash
curl -X POST http://localhost:8000/api/students \
  -H "Content-Type: application/json" \
  -d '{"name": "新しい生徒", "email": "new@example.com", "score": 90}'
```

**Windows（コマンドプロンプト）の場合：**
```cmd
curl -X POST http://localhost:8000/api/students -H "Content-Type: application/json" -d "{\"name\": \"新しい生徒\", \"email\": \"new@example.com\", \"score\": 90}"
```

> Windows では `"` を `\"` とエスケープする必要があります。

成功した場合のレスポンス（ステータスコード `201 Created`）：

```json
{"status":"ok","data":{"id":11,"name":"新しい生徒","email":"new@example.com","score":90,...}}
```

---

**② 更新（PUT /api/students/{id}）**

id が `1` の生徒の score を `95` に変更する例：

**Mac の場合：**
```bash
curl -X PUT http://localhost:8000/api/students/1 \
  -H "Content-Type: application/json" \
  -d '{"score": 95}'
```

**Windows（コマンドプロンプト）の場合：**
```cmd
curl -X PUT http://localhost:8000/api/students/1 -H "Content-Type: application/json" -d "{\"score\": 95}"
```

成功した場合のレスポンス（ステータスコード `200 OK`）：

```json
{"status":"ok","data":{"id":1,"name":"Tanaka Taro","email":"tanaka@example.com","score":95,...}}
```

---

**③ 削除（DELETE /api/students/{id}）**

id が `1` の生徒を削除する例：

**Mac の場合：**
```bash
curl -X DELETE http://localhost:8000/api/students/1
```

**Windows（コマンドプロンプト）の場合：**
```cmd
curl -X DELETE http://localhost:8000/api/students/1
```

成功した場合のレスポンス（ステータスコード `200 OK`）：

```json
{"status":"ok","message":"Deleted"}
```

---

**削除後に一覧を確認して、件数が減っていれば成功です：**

```bash
curl http://localhost:8000/api/students
```

### リクエストがどのファイルを通るか

`GET /api/students` をブラウザや curl で送ったとき、内部でどのファイルが順番に動いているのかを見ていきます。

---

#### 全体の流れ図

```
【クライアント（ブラウザ / curl）】
        |
        |  ① HTTP リクエスト送信
        |  GET http://localhost:8000/api/students
        ↓
┌─────────────────────────────────┐
│  routes/api.php                 │  ② どのコントローラに渡すか決める
│                                 │
│  Route::apiResource('students', │
│    StudentController::class);   │
└────────────────┬────────────────┘
                 |
                 |  ③ StudentController の index() を呼ぶ
                 ↓
┌─────────────────────────────────┐
│  app/Http/Controllers/Api/      │  ④ 処理を行う
│    StudentController.php        │
│                                 │
│  public function index() {      │
│    $students = Student::all();  │ ─────────────────┐
│    return response()->json(...)  │                  │ ⑤ DB問い合わせ
│  }                              │ ←────────────────┘
└────────────────┬────────────────┘
                 |  ⑥ Student::all() の中で
                 |     Eloquent ORM が SQL を自動生成・実行
                 ↓
┌─────────────────────────────────┐
│  app/Models/Student.php         │  ⑦ テーブルとの橋渡し役
│                                 │
│  class Student extends Model {} │
└────────────────┬────────────────┘
                 |
                 |  ⑧ SELECT * FROM students; を実行
                 ↓
┌─────────────────────────────────┐
│  データベース（MySQL）           │  ⑨ students テーブルからデータを取得
│  school_api.students テーブル   │
└────────────────┬────────────────┘
                 |
                 |  ⑩ 取得したデータを PHP の配列として返す
                 ↑  （ここから上向きに戻っていく）
┌─────────────────────────────────┐
│  StudentController.php          │  ⑪ 配列を JSON 形式に変換
```

#### 各ファイルの役割まとめ

| ファイル | 役割 | 担当するステップ |
|---------|------|---------------|
| `routes/api.php` | 「どの URL → どのコントローラ」かを対応づける | ② |
| `app/Http/Controllers/Api/StudentController.php` | リクエストを受け取り、処理して、レスポンスを返す | ④⑥⑪ |
| `app/Models/Student.php` | DB のテーブルと PHP コードの橋渡し | ⑦ |
| `データベース（MySQL）` | データを実際に保存・検索する | ⑨ |

---

#### Eloquent ORM とは（⑥⑦の補足）

`Student::all()` と書くだけで、Laravel が自動的に `SELECT * FROM students;` という SQL を生成して実行してくれます。  
この仕組みを **Eloquent ORM（エロクアント）** と言います。

```php
// PHP で書くこと        →  実際に実行される SQL
Student::all()           →  SELECT * FROM students;
Student::find(1)         →  SELECT * FROM students WHERE id = 1 LIMIT 1;
Student::create([...])   →  INSERT INTO students (...) VALUES (...);
$student->update([...])  →  UPDATE students SET ... WHERE id = ?;
$student->delete()       →  DELETE FROM students WHERE id = ?;
```

> **SQL を直接書かなくていい**のが ORM の利点です。  
> モデルのクラス名（`Student`）が自動的にテーブル名（`students`）に対応します。

---

#### POST のときの流れとの違い

`POST /api/students`（新規作成）の場合も基本の流れは同じですが、途中で**バリデーション**が入ります。

```
クライアント
    ↓ JSON データを送る（name, email, score）
routes/api.php  →  StudentController@store()
    ↓
$request->validate([...])   ← ここでデータの形式を検査
    ↓ NGなら 422 エラーを返す（ここで止まる）
    ↓ OKなら次へ進む
Student::create($request->all())  →  DB に INSERT
    ↓
JSON レスポンス（201 Created）を返す
    ↓
クライアント
```
