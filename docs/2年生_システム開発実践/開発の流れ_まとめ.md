# 開発の流れ：ゼロから API を作るまで

---

## はじめに

あなたたちは今学期、**1本のAPIをゼロから作りました**。

最初はGitのブランチから始まり、Laravelの環境を整え、データベースを作り、
エンドポイントを定義して、認証を追加し、最後にブラウザで動作を確認しました。

一つひとつの手順をこなすのに必死だったと思いますが、振り返ると
**5つのフェーズ**が一本の線でつながっています。

この資料では「何をしたか」だけでなく「**なぜその順番だったのか**」を解説します。

---

## 全体像：何を作ったのか

### 完成したシステムの全体図

```
[チームメンバー]
     │ git push / Pull Request
     ▼
[GitHub リポジトリ]
     │ clone / pull
     ▼
[Laravel API サーバー]  ←──  [認証トークン（Sanctum）]
     │                              ▲
     │ SQL クエリ              ログイン時に発行
     ▼
[MySQL データベース]
     ▲
     │ JSON レスポンス
[ブラウザ / React / スマホアプリ]
```

### 5つのフェーズ

| フェーズ | テーマ | キーワード | 担当授業 |
|---------|--------|-----------|---------|
| 1 | チーム開発の準備 | ブランチ・PR・レビュー | 04/30 |
| 2 | アプリの土台を作る | Laravel・DB・マイグレーション | 05/06 |
| 3 | データの窓口を作る | REST API・JSON・ルーティング | 05/14 |
| 4 | 形と守りを整える | Resource・バリデーション・Sanctum | 05/21 |
| 5 | 動いているか確かめる | curl・ブラウザテスト | 05/28 |
