# 新規ユーザー登録画面 作成の流れ

このドキュメントは、コマ23の課題「新規ユーザー登録画面を作る」を実際に手を動かして進めるための手順書です。  
`2026_07_09.md` で出されたお題を、これまでの授業（コマ17〜22）で作った `school-frontend` プロジェクトの上に実装していきます。

---

## 0. 全体像：何と何が繋がるのか

登録画面は「新しいレコードを1件作る」機能なので、CRUDでいう **Create** にあたります。  
流れはログイン画面（コマ17）とほぼ同じで、送り先のURLとバリデーション内容だけが変わります。

```
[Next.js] 登録フォームに入力
   名前・メール・パスワード・パスワード確認
        │ onSubmit
        ▼
   fetch("POST http://localhost:8000/api/register")
        │
        ▼
[Laravel] routes/api.php
   Route::post('/register', [AuthController::class, 'register']);
        │
        ▼
[Laravel] AuthController@register
   1. バリデーション（必須・emailの形式・emailの重複・パスワード一致）
   2. User::create() でDBに保存（パスワードはハッシュ化）
        │
        ▼
[MySQL] users テーブルに1件追加
        │
        ▼
[Laravel] レスポンスを返す（成功 or 422エラー）
        │
        ▼
[Next.js] 成功なら /login へ移動、失敗ならエラー文言を表示
```

`routes/api.php` には、実は **コマ08の時点ですでに登録用のルートが書かれています。**

```php
// routes/api.php（すでに書いてあるはず）
Route::post('/login', [AuthController::class, 'login']);
Route::post('/register', [AuthController::class, 'register']);
```

ただし `AuthController` の中身は `login` / `logout` / `me` しか実装していないので、  
**`register()` メソッドが存在しない状態**です。今日はこれを作るところから始めます。

## 1. Laravel側：`register()` を実装する

`app/Http/Controllers/Api/AuthController.php` を開き、`login()` メソッドの近くに追加します。

```php
// app/Http/Controllers/Api/AuthController.php
use App\Models\User;
use Illuminate\Support\Facades\Hash;

// POST /api/register
public function register(Request $request)
{
    // ① 入力チェック
    $validated = $request->validate([
        'name'     => 'required|string|max:255',
        'email'    => 'required|email|unique:users,email',
        'password' => 'required|min:8|confirmed',
    ]);

    // ② ユーザーを作成（パスワードは必ずハッシュ化する）
    $user = User::create([
        'name'     => $validated['name'],
        'email'    => $validated['email'],
        'password' => Hash::make($validated['password']),
    ]);

    // ③ そのままログインさせたいので、トークンも発行して返す
    $token = $user->createToken('api-token')->plainTextToken;

    return response()->json([
        'status' => 'ok',
        'token'  => $token,
        'user'   => $user,
    ], 201); // 201 = 新規作成に成功
}
```

### バリデーションのポイント

| ルール | 意味 |
|--------|------|
| `required` | 未入力なら422エラー |
| `email` | メール形式かチェック |
| `unique:users,email` | `users`テーブルに同じメールが無いか確認（重複登録防止） |
| `min:8` | パスワードは8文字以上 |
| `confirmed` | **「同じ名前 + `_confirmation`」のフィールドと値が一致するか**を自動チェックする特別なルール |

`confirmed` ルールを使うと、フロントエンドから送るキー名は必ず次のようにする必要があります。

```json
{
  "name": "田中太郎",
  "email": "tanaka@example.com",
  "password": "password123",
  "password_confirmation": "password123"
}
```

`password_confirmation` という名前を1文字でも間違えると、`confirmed` ルールが効かず「一致しているのにエラーになる」バグの原因になるので注意してください。

## 2. Next.js側：`/register` ページを作る

`app/register/` フォルダを作成し、`page.tsx` を新規作成します。

```
app/register/page.tsx  ← 新規作成
```

ログインページ（コマ17）と同じ構造で作ります。**違うのは入力項目と送信先だけ**です。

```tsx
"use client";

import { useState } from "react";
import { useRouter } from "next/navigation";

export default function RegisterPage() {
  const router = useRouter();
  const [name, setName] = useState("");
  const [email, setEmail] = useState("");
  const [password, setPassword] = useState("");
  const [passwordConfirmation, setPasswordConfirmation] = useState("");
  const [error, setError] = useState("");
  const [loading, setLoading] = useState(false);

  async function handleSubmit(e) {
    e.preventDefault();
    setError("");
    setLoading(true);

    const res = await fetch("http://localhost:8000/api/register", {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        "Accept": "application/json",
      },
      // ★ キー名はLaravel側のバリデーションと合わせる
      body: JSON.stringify({
        name,
        email,
        password,
        password_confirmation: passwordConfirmation,
      }),
    });

    const data = await res.json();
    setLoading(false);

    if (res.ok) {
      // ★ そのままログイン状態にする（トークンを保存して/studentsへ）
      localStorage.setItem("token", data.token);
      localStorage.setItem("userName", data.user?.name || "ユーザー");
      router.push("/students");
    } else if (data.errors) {
      // コマ21で学んだのと同じ、Laravelのバリデーションエラー表示
      const messages = Object.values(data.errors).flat().join(" / ");
      setError(messages);
    } else {
      setError(`エラーが発生しました（HTTP ${res.status}）`);
    }
  }

  return (
    <div style={{
      maxWidth: "400px", margin: "80px auto",
      background: "white", padding: "32px",
      borderRadius: "8px", boxShadow: "0 2px 8px rgba(0,0,0,0.1)"
    }}>
      <h2 style={{ marginBottom: "24px", textAlign: "center" }}>新規ユーザー登録</h2>

      <form onSubmit={handleSubmit}>
        <div style={{ marginBottom: "16px" }}>
          <label style={{ display: "block", marginBottom: "6px", fontWeight: "bold" }}>名前</label>
          <input
            type="text"
            value={name}
            onChange={(e) => setName(e.target.value)}
            style={{ width: "100%", padding: "10px", border: "1px solid #ccc", borderRadius: "6px", fontSize: "1em", boxSizing: "border-box" }}
          />
        </div>

        <div style={{ marginBottom: "16px" }}>
          <label style={{ display: "block", marginBottom: "6px", fontWeight: "bold" }}>メールアドレス</label>
          <input
            type="email"
            value={email}
            onChange={(e) => setEmail(e.target.value)}
            style={{ width: "100%", padding: "10px", border: "1px solid #ccc", borderRadius: "6px", fontSize: "1em", boxSizing: "border-box" }}
          />
        </div>

        <div style={{ marginBottom: "16px" }}>
          <label style={{ display: "block", marginBottom: "6px", fontWeight: "bold" }}>パスワード</label>
          <input
            type="password"
            value={password}
            onChange={(e) => setPassword(e.target.value)}
            style={{ width: "100%", padding: "10px", border: "1px solid #ccc", borderRadius: "6px", fontSize: "1em", boxSizing: "border-box" }}
          />
        </div>

        <div style={{ marginBottom: "16px" }}>
          <label style={{ display: "block", marginBottom: "6px", fontWeight: "bold" }}>パスワード確認</label>
          <input
            type="password"
            value={passwordConfirmation}
            onChange={(e) => setPasswordConfirmation(e.target.value)}
            style={{ width: "100%", padding: "10px", border: "1px solid #ccc", borderRadius: "6px", fontSize: "1em", boxSizing: "border-box" }}
          />
        </div>

        {error && (
          <p style={{ color: "#D32F2F", fontSize: "0.9em", marginBottom: "12px" }}>
            {error}
          </p>
        )}

        <button
          type="submit"
          disabled={loading}
          style={{ width: "100%", padding: "12px", background: loading ? "#aaa" : "#1976D2", color: "white", border: "none", borderRadius: "6px", fontSize: "1em", fontWeight: "bold", cursor: loading ? "not-allowed" : "pointer" }}
        >
          {loading ? "登録中..." : "登録する"}
        </button>

        <p style={{ textAlign: "center", marginTop: "16px", fontSize: "0.9em" }}>
          すでにアカウントをお持ちの方は{" "}
          <a href="/login" style={{ color: "#1976D2", fontWeight: "bold" }}>こちら</a>
        </p>
      </form>
    </div>
  );
}
```

### ログインページ（コマ17）との違い

| | `/login`（コマ17） | `/register`（今回） |
|--|--|--|
| 入力項目 | メール・パスワード | 名前・メール・パスワード・パスワード確認 |
| 送信先 | `POST /api/login` | `POST /api/register` |
| 成功時のレスポンス | `{ token, user }` | `{ token, user }`（同じ形） |
| 失敗時の表示 | 固定文言（401） | Laravelのバリデーション文言（422） |
| 成功後の遷移 | `/students` | `/students`（登録と同時にログイン扱いにする） |

## 3. ログイン画面と登録画面を相互にリンクする

課題の仕様（`2026_07_09.md`）にある通り、登録画面には「すでにアカウントをお持ちの方はこちら」というログイン画面へのリンクが必要です（上記コードの末尾にすでに追加済み）。

あわせて、ログイン画面側にも登録画面へのリンクを追加しておくと、ユーザーが迷わず行き来できます。

```tsx
// app/login/page.tsx の <form> の一番下に追加
<p style={{ textAlign: "center", marginTop: "16px", fontSize: "0.9em" }}>
  アカウントをお持ちでない方は{" "}
  <a href="/register" style={{ color: "#1976D2", fontWeight: "bold" }}>こちら</a>
</p>
```

## 4. 両サーバーを起動して動作確認する

```bash
# ターミナル① Laravel
cd （Laravelプロジェクトのフォルダ）
php artisan serve

# ターミナル② Next.js
cd school-frontend
npm run dev
```

### 動作確認チェックリスト

| 確認項目 | 操作 | 期待する結果 |
|---------|------|------------|
| 画面表示 | `http://localhost:3000/register` を開く | 名前・メール・パスワード・パスワード確認の入力欄が表示される |
| 正常登録 | すべて正しく入力して「登録する」を押す | `/students` に移動し、ログイン済み状態になる |
| メール重複 | 既に登録済みのメールアドレスで登録 | 「The email has already been taken.」などのエラーが表示される |
| パスワード不一致 | パスワードとパスワード確認を違う値にする | 「The password confirmation does not match.」が表示される |
| 未入力 | 何も入力せず送信 | 各項目の「required」エラーが表示される |
| 画面遷移（登録→ログイン） | 「すでにアカウントをお持ちの方はこちら」を押す | `/login` に移動する |
| 画面遷移（ログイン→登録） | ログイン画面の「こちら」を押す | `/register` に移動する |
| ログイン確認 | 新規登録したメール・パスワードでログインし直す | ログインに成功する |
