## Laravel プロジェクトの初期構築

> **目的：** この授業で使う Laravel プロジェクトをゼロから作り、API 開発を始められる状態にする。

---

### はじめに：必要なものが入っているか確認する

作業を始める前に、必要なソフトがインストールされているか確認します。  
ターミナルを開いて、以下のコマンドを**1行ずつ**実行してください。

---

#### PHP の確認

```bash
php --version
```

**OK の場合：**
```
PHP 8.2.x (cli) ...  ← バージョンが表示されればOK
```

**NG の場合（`command not found` または `'php' は認識されていません` と表示された）：**

PHP が入っていません。XAMPP をインストールすると PHP と MySQL をまとめて使えるようになります。

**Windows の場合：**
1. `https://www.apachefriends.org/` にアクセスする
2. 「XAMPP for Windows」をダウンロードしてインストールする
3. インストール後、`C:\xampp\php` を**環境変数 PATH** に追加する  
   （スタートメニューで「環境変数」と検索 → 「システム環境変数の編集」→「環境変数」→「Path」を選んで「編集」→「新規」で `C:\xampp\php` を追加）
4. ターミナルを**閉じて開き直し**、もう一度 `php --version` を実行する

**Mac の場合：**
1. ターミナルで以下を実行する（Homebrew がある場合）
   ```bash
   brew install php
   ```
2. Homebrew が入っていない場合はまず以下を実行する
   ```bash
   /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
   ```

---

#### Composer の確認

```bash
composer --version
```

**OK の場合：**
```
Composer version 2.x.x ...  ← バージョンが表示されればOK
```

**NG の場合（`command not found` または `'composer' は認識されていません` と表示された）：**

Composer が入っていません。以下の手順でインストールします。

**Windows の場合：**
1. `https://getcomposer.org/download/` にアクセスする
2. 「Composer-Setup.exe」をダウンロードして実行する
3. インストール中に PHP の場所を聞かれたら `C:\xampp\php\php.exe` を指定する
4. インストール完了後、ターミナルを**閉じて開き直し**、`composer --version` を実行する

**Mac の場合：**
```bash
brew install composer
```

---

#### MySQL の確認

```bash
mysql --version
```

**OK の場合：**
```
mysql  Ver 8.0.x ...  ← バージョンが表示されればOK
```

バージョンが表示されたら、次に MySQL **サーバー**が起動しているかを確認します。

> **注意（Mac）：** `mysql --version` が表示されても、サーバーが入っていない場合があります。  
> `mysql-client`（接続ツールのみ）と `mysql`（サーバー本体）は別物です。  
> 以下の手順でサーバーが動いているか必ず確認してください。

**Macの場合：**
```bash
brew services list | grep mysql
```

| 表示 | 意味 |
|------|------|
| `mysql  started` | サーバーが起動中。次のステップへ進んでOK |
| `mysql  stopped` | サーバーが停止中。下の「起動する」手順へ |
| mysql の行が出ない | サーバー本体が未インストール。下の「インストールする」手順へ |

**Windows（XAMPP）の場合：**  
XAMPP コントロールパネルを開いて、MySQL の横の「Start」ボタンを押します。  
「Running」と表示されれば起動済みです。

**MySQL サーバーが stopped の場合（起動する）：**

```bash
# Mac の場合
brew services start mysql
```

XAMPP の場合はコントロールパネルの「Start」ボタンを押します。

**MySQL サーバー本体が未インストールの場合（Mac）：**

```bash
brew install mysql
brew services start mysql
```

> **接続時のパスワードについて**  
> `mysql -u root -p` を実行するとパスワードを聞かれます。  
> Homebrew で新規インストールした場合は**パスワードなし**なのでそのまま Enter を押します。  
> XAMPP の場合も初期設定では**パスワードなし**です。  
> 自分でパスワードを設定した覚えがある場合はそのパスワードを入力します。

---

**NG の場合（`command not found` または `'mysql' は認識されていません` と表示された）：**

MySQL が入っていません。

**Windows（XAMPP を使う場合）：**
1. XAMPP がインストール済みであれば、`C:\xampp\mysql\bin` を環境変数 PATH に追加する  
   （PHP の追加と同じ手順で、今度は `C:\xampp\mysql\bin` を追加する）
2. ターミナルを**閉じて開き直し**、`mysql --version` を実行する

**Mac の場合：**
```bash
brew install mysql
brew services start mysql
```

---

> **それでも解決しない場合は先生に声をかけてください。**  
> バージョンが古すぎる（PHP 7系など）場合も先生に確認しましょう。  
> **PHP 8.1 以上・Composer 2系・MySQL 8系** が推奨バージョンです。

### Step 1｜クローンしたリポジトリの中に移動する

すでに GitHub からクローンしたリポジトリがあるはずです。  
ターミナルを開いて、そのフォルダの中に移動します。

**Windowsの場合（PowerShellまたはコマンドプロンプト）：**
```bash
cd C:\Users\（自分のユーザー名）\Documents\（リポジトリ名）
```

**Macの場合：**
```bash
cd ~/Documents/（リポジトリ名）
```

移動できたか確認します：
```bash
pwd
```
```
/Users/（自分のユーザー名）/Documents/（リポジトリ名）  ← リポジトリの中にいればOK
```

Git リポジトリとして認識されているかも確認しましょう：
```bash
git status
```
```
On branch main
nothing to commit, working tree clean  ← このように表示されればOK
```

> **`git status` でエラーが出た場合**（`not a git repository` など）  
> リポジトリの中に移動できていません。フォルダ名を確認して、もう一度 `cd` し直してください。

---

### Step 2｜Laravel をインストールする

クローンしたフォルダには `git clone` によって自動生成される `.git` フォルダが存在します。  
`composer create-project laravel/laravel .` はフォルダが完全に空でないと実行できないため、エラーになります。

```
Project directory "..." is not empty.  ← このエラーが出る
```

> **`.git` は隠しフォルダのため、Finder やエクスプローラーでは見えません。**  
> ターミナルで `ls -a` を実行すると確認できます。

そのため、**いったん別のフォルダに作ってから中身を移動する**手順で進めます。

#### 2-1. 1つ上のフォルダに移動する

```bash
cd ..
```

確認します：
```bash
pwd
```
```
/Users/（自分のユーザー名）/Documents  ← リポジトリの1つ上にいればOK
```

#### 2-2. 一時フォルダに Laravel をインストールする

```bash
composer create-project laravel/laravel temp-laravel
```

> 実行には **2〜5分** かかります。画面にたくさん文字が流れますが、待ちましょう。  
> 最後に `Application ready!` と表示されれば成功です。

#### 2-3. 一時フォルダの中身をリポジトリフォルダに移動する

**Macの場合：**
```bash
cp -r temp-laravel/. （リポジトリ名）/
rm -rf temp-laravel
```

**Windowsの場合（PowerShell）：**
```powershell
Copy-Item -Path temp-laravel\* -Destination （リポジトリ名）\ -Recurse -Force
Copy-Item -Path temp-laravel\.* -Destination （リポジトリ名）\ -Recurse -Force -ErrorAction SilentlyContinue
Remove-Item -Recurse -Force temp-laravel
```

> **（リポジトリ名）** の部分は自分のリポジトリのフォルダ名に置き換えてください。

#### 2-4. リポジトリフォルダに戻る

```bash
cd （リポジトリ名）
```

正しく移動できたか確認します：
```bash
pwd
```
```
/Users/（自分のユーザー名）/Documents/（リポジトリ名）  ← リポジトリの中にいればOK
```

`app/` や `routes/` などのフォルダが作られていれば成功です：
```bash
ls
```
```
README.md  app  artisan  bootstrap  composer.json  config  database  ...
```

#### 2-5. アプリケーションキーを生成する

Laravel はセッションや暗号化のために**アプリケーションキー**（秘密鍵）を必要とします。  
必ず実行してください。

```bash
php artisan key:generate
```

```
   INFO  Application key set successfully.
```

> このコマンドを実行すると、`.env` ファイルの `APP_KEY=` に値が自動で設定されます。  
> 忘れると `MissingAppKeyException` というエラー画面が表示されます。

**エラーが出た場合：`No APP_KEY variable was found in the .env file`**

`.env` ファイルに `APP_KEY=` の行がありません。  
VSCode で `.env` を開き、ファイルの末尾に以下の1行を追記してください。

```dotenv
APP_KEY=
```

保存したら、もう一度実行します：

```bash
php artisan key:generate
```

`APP_KEY=` の後に長い文字列が自動で入れば成功です。

**この時点でのフォルダ構成（主要なものだけ）：**

```
（リポジトリ名）/          ← クローンしたフォルダ（= Git リポジトリ）
├── app/
│   ├── Http/
│   │   └── Controllers/   ← コントローラを置く場所
│   └── Models/            ← モデルを置く場所
├── database/
│   ├── migrations/        ← テーブル定義を置く場所
│   ├── factories/         ← テストデータ生成の設定
│   └── seeders/           ← テストデータ投入の設定
├── routes/
│   ├── api.php            ← APIのルーティング（今回メインで使う）
│   └── web.php            ← Bladeのルーティング（1年生で使ったもの）
└── .env                   ← DB接続情報などの設定ファイル
```

### Step 3｜データベースを用意する

#### phpMyAdmin とは？

**phpMyAdmin** は、MySQL のデータベースをブラウザ上で操作できる管理ツールです。  
コマンドを打たなくても、画面を操作するだけでデータベースの作成・確認・削除ができます。

| 環境             | phpMyAdmin の開き方                                                        |
| -------------- | ---------------------------------------------------------------------- |
| Windows（XAMPP） | XAMPPコントロールパネルの「Admin」ボタンを押す → ブラウザで `http://localhost/phpmyadmin` が開く |
| Mac（XAMPP）     | XAMPPコントロールパネルの「Admin」ボタンを押す → ブラウザで `http://localhost/phpmyadmin` が開く |
| Mac（Homebrew）  | phpMyAdmin は別途インストールが必要（後述のターミナル手順を使う方が簡単）                             |

---

#### 3-1. データベースを作成する

**Windowsの場合（XAMPP + phpMyAdmin）：**

1. XAMPP コントロールパネルで MySQL を「Start」する
2. ブラウザで `http://localhost/phpmyadmin` を開く
3. 左側メニューの「**新規作成**」をクリックする
4. 以下を入力して「**作成**」ボタンを押す

   - **データベース名：** `school_api`
   - **照合順序：** `utf8mb4_unicode_ci`

**Macの場合（Homebrew）：**

phpMyAdmin のセットアップが不要なターミナルから作成します。  
MySQL が起動していることを確認してから実行してください。

```bash
mysql -u root
```
```sql
CREATE DATABASE school_api CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
EXIT;
```

> パスワードを設定している場合は `mysql -u root -p` と入力し、パスワードを入力してください。

#### 3-2. `.env` ファイルを設定する

VSCode でプロジェクトを開き、ルート直下にある `.env` ファイルを開きます。

```
（リポジトリ名）/
└── .env   ← これを開く
```

> **⚠️ 注意：ファイルの内容を全部消して書き直さないでください。**  
> `.env` にはすでに `APP_KEY` など多くの設定が書かれています。  
> **DB に関係する5行だけを探して値を書き換えます。それ以外の行は触らないでください。**

ファイルの中を下にスクロールすると、以下のような行があります。  
この5行の `=` の右側だけを書き換えます：

```dotenv
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=school_api    ← 先ほど作ったDBの名前に変える
DB_USERNAME=root           ← MySQLのユーザー名（通常は root）
DB_PASSWORD=               ← パスワードを設定していなければ空欄のまま
```

書き換えたら **Ctrl+S**（Mac は **Cmd+S**）で保存します。

> **`.env` はパスワードが書かれた設定ファイルです。**  
> Git には自動的に含まれないようになっているので、他人に見せないようにしましょう。

### Step 4｜モデル・マイグレーション・ファクトリを作る

この授業では **生徒（Student）** を管理する API を作ります。  
以下の1コマンドで、必要なファイルを一括作成できます。

```bash
php artisan make:model Student -mf
```

| オプション | 意味                  | 作られるファイル                                             |
| ----- | ------------------- | ---------------------------------------------------- |
| （なし）  | モデルを作る              | `app/Models/Student.php`                             |
| `-m`  | マイグレーション（テーブル定義）も作る | `database/migrations/xxxx_create_students_table.php` |
| `-f`  | ファクトリ（テストデータ生成）も作る  | `database/factories/StudentFactory.php`              |

実行後に以下のように表示されれば成功です：

```
   INFO  Model [app/Models/Student.php] created successfully.
   INFO  Factory [database/factories/StudentFactory.php] created successfully.
   INFO  Migration [database/migrations/2024_xx_xx_create_students_table.php] created successfully.
```

---

### Step 5｜モデルを編集する（`$fillable` の設定）

作成された `app/Models/Student.php` を VSCode で開きます。

```
school-api/
└── app/
    └── Models/
        └── Student.php   ← これを開く
```

ファイルを開くと以下のような内容になっています。`$fillable` を追加します：

```php
<?php
namespace App\Models;

use Illuminate\Database\Eloquent\Factories\HasFactory;
use Illuminate\Database\Eloquent\Model;

class Student extends Model
{
    use HasFactory;

    // ↓ この1行を追加する
    protected $fillable = ['name', 'email', 'score'];
}
```

> **`$fillable` とは？**  
> Laravel のセキュリティ機能です。書いたカラム名だけを `create()` や `update()` で変更できるようになります。  
> これを設定しておかないと、後で API からデータを保存するときにエラーになります。

---

### Step 6｜マイグレーションファイルを編集する

`database/migrations/` フォルダを VSCode で開くと、日付から始まるファイルが並んでいます。

```
school-api/
└── database/
    └── migrations/
        ├── 0001_01_01_000000_create_users_table.php    ← Laravel が最初から用意しているもの
        ├── 0001_01_01_000001_create_cache_table.php    ← Laravel が最初から用意しているもの
        ├── 0001_01_01_000002_create_jobs_table.php     ← Laravel が最初から用意しているもの
        └── 2024_xx_xx_xxxxxx_create_students_table.php ← ← Step4 で作られたもの（これを編集）
```

**一番下の `create_students_table.php` を開き**、`up()` メソッドの中身を書き換えます：

```php
public function up(): void
{
    Schema::create('students', function (Blueprint $table) {
        $table->id();                              // ID（自動で連番）
        $table->string('name', 100);              // 氏名
        $table->string('email')->unique();         // メールアドレス（重複不可）
        $table->integer('score')->default(0);     // 点数（初期値は0）
        $table->timestamps();                      // 作成日時・更新日時（自動）
    });
}
```

#### よくあるエラーと対処法

**エラー1：DB接続ができない**
```
SQLSTATE[HY000] [2002] Connection refused
```
→ MySQL が起動していません。  
→ **Windows：** XAMPP コントロールパネルで MySQL を「Start」してください。  
→ **Mac：** `brew services start mysql` を実行してください。

**エラー2：データベースが見つからない**
```
SQLSTATE[HY000] [1049] Unknown database 'school_api'
```
→ Step3 でデータベースを作成していないか、`.env` のデータベース名が違います。  
→ **Windows：** phpMyAdmin でデータベース名を確認してください。  
→ **Mac：** `mysql -u root -e "SHOW DATABASES;"` でデータベース一覧を確認してください。

**エラー3：パスワードが違う**
```
SQLSTATE[HY000] [1045] Access denied for user 'root'@'localhost'
```
→ `.env` の `DB_PASSWORD` が間違っています。MySQL のパスワードを確認して書き直してください。

### Step 8｜テストデータを投入する

API の動作確認に使うテストデータを用意します。

#### 8-1. ファクトリを編集する

`database/factories/StudentFactory.php` を開きます。

```
school-api/
└── database/
    └── factories/
        └── StudentFactory.php   ← これを開く
```

`definition()` メソッドの中身を書き換えます：

```php
public function definition(): array
{
    return [
        'name'  => $this->faker->name(),                      // ランダムな名前
        'email' => $this->faker->unique()->safeEmail(),        // ランダムなメール
        'score' => $this->faker->numberBetween(0, 100),        // 0〜100のランダムな点数
    ];
}
```

#### 8-2. シーダーを編集する

`database/seeders/DatabaseSeeder.php` を開き、`run()` メソッドに1行追加します：

```php
public function run(): void
{
    \App\Models\Student::factory(10)->create(); // 生徒データを10件作成
}
```

#### 8-3. テストデータを投入する

```bash
php artisan db:seed
```

```
   INFO  Seeding database.
```

データが入ったか確認します。

**Windowsの場合（phpMyAdmin）：**  
`http://localhost/phpmyadmin` を開き、`school_api` → `students` をクリックすると10件のデータが表示されます。

**Macの場合（ターミナル）：**
```bash
mysql -u root -e "SELECT COUNT(*) FROM school_api.students;"
```
```
+----------+
| COUNT(*) |
+----------+
|       10 |  ← 10 と表示されればOK
+----------+
```

### Step 9｜開発サーバーを起動する

```bash
php artisan serve
```

```
   INFO  Server running on [http://127.0.0.1:8000].

  Press Ctrl+C to stop the server
```

ブラウザで `http://localhost:8000` にアクセスして、「Laravel」のロゴが表示されれば起動成功です。

> **このトップ画面はサーバーが動いていることを確認するだけの画面です。**  
> シーダーで入れたデータはデータベースの中にあり、この画面には表示されません。  
> データは次のコマから API エンドポイントを作ることで取得・表示できるようになります。

> **サーバーを止めるときは `Ctrl + C` を押します。**  
> サーバーが動いている間はターミナルを閉じないでください。

#### シーダーデータが正しく入っているか確認したい場合

**Macの場合（ターミナルで確認）：**

```bash
mysql -u root -e "SELECT id, name, score FROM school_api.students LIMIT 5;"
```

```
+----+------------------+-------+
| id | name             | score |
+----+------------------+-------+
|  1 | Tanaka Taro      |    72 |
|  2 | Yamada Hanako    |    88 |
...
+----+------------------+-------+
```

このようにランダムなデータが表示されればシーダーは成功しています。

**Windowsの場合（phpMyAdmin で確認）：**

`http://localhost/phpmyadmin` を開き、`school_api` → `students` をクリックすると、10件のデータが一覧で表示されます。
