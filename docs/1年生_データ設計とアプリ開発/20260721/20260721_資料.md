## 1. 配列だけでデータを管理してみる

まず、PHP の配列だけで「ユーザー管理システム」を作ってみましょう。

```php
<?php
// ファイルで状態を保存しようとする試み
// users.php
$users = [
    ["id" => 1, "name" => "田中太郎", "email" => "tanaka@example.com"],
    ["id" => 2, "name" => "鈴木花子", "email" => "suzuki@example.com"],
];

// 新しいユーザーを追加する関数
function addUser(array &$users, string $name, string $email): void {
    $newId = count($users) + 1;
    $users[] = ["id" => $newId, "name" => $name, "email" => $email];
}

addUser($users, "山田次郎", "yamada@example.com");

// 表示
foreach ($users as $user) {
    echo "{$user['id']}: {$user['name']}\n";
}
```

### 問題点1：サーバーを再起動するとデータが消える

```
PHP のプログラムは、リクエストが来るたびに最初から実行される
 → 毎回 $users = [] から始まる
 → 前回追加したデータは消えてしまう！
```

### 問題点2：複数の人が同時にアクセスするとデータが壊れる

```
ユーザーA: ファイルを読む → データ追加 → ファイルに書く
ユーザーB:          ファイルを読む → データ追加 → ファイルに書く
                                          ↑ Aの追加が上書きされて消える！
```

### 問題点3：データが増えると検索が遅い

```php
// 100万人のユーザーから特定のメールアドレスを探す
foreach ($users as $user) {
    if ($user["email"] === "target@example.com") {
        return $user;  // 最悪100万回ループする...
    }
}
```

## 2. データベースが解決すること

```
┌───────────────────────────────────────────────┐
│  データベース（MySQL など）                     │
│                                               │
│  ✅ データを「永続的に」保存する               │
│     → サーバーを再起動してもデータは残る       │
│                                               │
│  ✅ 同時アクセスを「安全に」処理する           │
│     → 複数人が同時に書き込んでもデータが壊れない│
│                                               │
│  ✅ 大量データを「高速に」検索できる           │
│     → インデックスという仕組みで超高速検索     │
│                                               │
│  ✅ データの「整合性」を守る                   │
│     → 不正なデータが入らないようにルールを設定 │
└───────────────────────────────────────────────┘
```

## 3. データベースの基本概念

データベースは**スプレッドシートに似た構造**で考えると分かりやすいです。

```
データベース（ファイル全体）
  └── テーブル（シート1枚）
        ├── カラム（列：名前・年齢・メールなど）
        └── レコード（行：1人分のデータ）

例：users テーブル

 id │ name     │ email                │ created_at
────┼──────────┼──────────────────────┼─────────────────
  1 │ 田中太郎 │ tanaka@example.com   │ 2024-01-01 10:00
  2 │ 鈴木花子 │ suzuki@example.com   │ 2024-01-02 11:00
  3 │ 山田次郎 │ yamada@example.com   │ 2024-01-03 12:00
```

## 4. MySQL の基本

**MySQL = 世界で最も使われるオープンソースのリレーショナルデータベース**

Laravel でも標準的に使います。

```
リレーショナルデータベース（RDB）とは：
- データをテーブル（表）で管理する
- テーブル同士を「関係（リレーション）」で結ぶことができる
- SQL（Structured Query Language）という言語で操作する

例：
  users テーブル（ユーザー情報）
  posts テーブル（投稿情報）
  → posts.user_id でどのユーザーの投稿かを管理できる
```

## 5. MySQL を用意する（Windows: XAMPP / Mac: DBngin）

自分のOSに合わせて、どちらか一方を使います。

### Windows の場合：XAMPP

**XAMPP = Apache / MySQL（MariaDB）/ PHP / phpMyAdmin をまとめてインストールできる無料ツール**

#### インストール

[apachefriends.org](https://www.apachefriends.org/) からダウンロードしてインストール

#### MySQL サーバーの起動

1. XAMPP Control Panel を起動
2. 「MySQL」の行にある「Start」をクリックして起動

### Mac の場合：DBngin

**DBngin = MySQL / PostgreSQL / Redis をワンクリックで起動できる無料ツール**

#### インストール

[dbngin.com](https://dbngin.com/) からダウンロードしてインストール

#### MySQL サーバーの作成・起動

1. DBngin を起動
2. 「+」ボタン →「MySQL」を選択
3. バージョンを選んで「Create」（バージョンはどれでもOK、迷ったら最新の安定版）
4. 作成したサーバーの「Start」をクリックして起動

### 共通の接続情報

どちらのツールでも、デフォルトでは以下の設定で MySQL が立ち上がります。

```
Host:     127.0.0.1
Port:     3306
User:     root
Password: （空でOK）
```

※ すでに 3306 番ポートを別のMySQLが使っている場合は、ツール側で別のポート番号に変更してください（その場合はGUIツールの接続設定もそのポート番号に合わせます）。

## 6. GUI ツールで MySQL を操作する（Windows: phpMyAdmin / Mac: TablePlus）

### Windows の場合：phpMyAdmin

**phpMyAdmin = XAMPPに標準で付属しているMySQL管理画面（ブラウザから操作）**

XAMPP Control Panel で MySQL を Start した状態で、ブラウザから以下にアクセスします。

```
http://localhost/phpmyadmin/
```

接続情報の入力は不要です（XAMPP起動時にrootユーザーで自動接続されます）。

### Mac の場合：TablePlus

**TablePlus = MySQL を GUI で操作できるツール（手書きSQLが練習できる）**

#### インストール

[tableplus.com](https://tableplus.com/) からダウンロード

#### 接続設定（DBngin で起動した MySQL に接続する場合）

```
Name:     local
Host:     127.0.0.1
Port:     3306
User:     root
Password: （空でOK）
Database: （後で作成）
```

## 7. 最初のデータベースを作る

Windows は phpMyAdmin の「SQL」タブ、Mac は TablePlus の SQL タブで以下を実行：

```sql
-- データベースを作成する
CREATE DATABASE school_db;

-- 作成したデータベースを使う
USE school_db;

-- テーブルを作成する
CREATE TABLE users (
    id    INT AUTO_INCREMENT PRIMARY KEY,  -- 自動で番号が付く
    name  VARCHAR(100) NOT NULL,           -- 最大100文字の文字列（必須）
    email VARCHAR(255) NOT NULL UNIQUE,    -- UNIQUE = 重複禁止
    age   INT,                             -- 整数（任意）
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP  -- 作成日時
);

-- データを入れてみる
INSERT INTO users (name, email, age) VALUES ('田中太郎', 'tanaka@example.com', 20);
INSERT INTO users (name, email, age) VALUES ('鈴木花子', 'suzuki@example.com', 22);

-- データを取り出す
SELECT * FROM users;
```

## 番外編：PHP から SQL を実行している例文

PHP には **PDO（PHP Data Objects）** という、DB接続用の標準機能があります。

```php
<?php
$pdo = new PDO(
    'mysql:host=127.0.0.1;port=3306;dbname=school_db;charset=utf8mb4',
    'root',
    '' // XAMPP/DBngin のデフォルトは空パスワード
);

// 実行するSQL文
$sql = "SELECT * FROM users";

$stmt = $pdo->query($sql);
$users = $stmt->fetchAll(PDO::FETCH_ASSOC);

foreach ($users as $user) {
    echo "{$user['id']}: {$user['name']}\n";
}
```
