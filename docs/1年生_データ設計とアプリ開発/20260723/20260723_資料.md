# コマ08｜SQL基礎② — INSERT / UPDATE / DELETE

## 今日のゴール
- INSERT でデータを追加できる
- UPDATE でデータを更新できる
- DELETE でデータを削除できる
- WHERE の重要性（つけ忘れると全件変更になる）を理解している

---

## 1. INSERT — データを追加する

```sql
-- 全カラムに値を入れる場合（カラム名を省略できる）
INSERT INTO students VALUES (NULL, '新入生A', 18, 'A', 80, '2024-04-01');
-- ↑ id は AUTO_INCREMENT なので NULL でOK

-- 特定のカラムだけ指定して入れる（推奨）
INSERT INTO students (name, age, class, score, enrolled_at)
VALUES ('新入生B', 19, 'B', 75, '2024-04-01');

-- 複数行を一度に入れる
INSERT INTO students (name, age, class, score, enrolled_at)
VALUES
    ('新入生C', 18, 'C', 88, '2024-04-01'),
    ('新入生D', 20, 'A', 62, '2024-04-01'),
    ('新入生E', 19, 'B', 71, '2024-04-01');
```

---

## 2. UPDATE — データを更新する

```sql
-- !! 超重要 !!
-- WHERE を付け忘れると全行が更新されてしまう！
-- 必ず WHERE を確認してから実行する！

-- id が 1 の生徒の score を 90 に更新
UPDATE students SET score = 90 WHERE id = 1;

-- 複数カラムを同時に更新（カンマ区切り）
UPDATE students SET score = 95, class = 'S' WHERE id = 1;

-- WHERE なし（危険！全行が更新される）
-- UPDATE students SET score = 0;  ← これをやると全員0点になる！

-- 現在の値を使って更新（例：全員に+5点）
UPDATE students SET score = score + 5;  -- これは意図的なら良い
```

---

## 3. DELETE — データを削除する

```sql
-- !! 超重要 !!
-- WHERE を付け忘れると全行が削除されてしまう！
-- DELETE は元に戻せない！

-- id が 5 の生徒を削除
DELETE FROM students WHERE id = 5;

-- score が 50 未満の生徒を全員削除
DELETE FROM students WHERE score < 50;

-- WHERE なし（絶対にやってはいけない）
-- DELETE FROM students;  ← テーブルが空になる！
```

> **実務での鉄則：**  
> 1. UPDATE/DELETE の前に SELECT で対象を確認する  
> 2. 実行前にバックアップを取る  
> 3. WHERE を必ず確認する

---

## 4. 安全な UPDATE/DELETE のやり方

```sql
-- ステップ1：まず SELECT で対象を確認
SELECT * FROM students WHERE score < 50;
-- → 削除される行を目で確認する

-- ステップ2：問題なければ DELETE を実行
DELETE FROM students WHERE score < 50;

-- 確認
SELECT * FROM students;
```

---

## 5. よく使うパターン：UPSERT

```sql
-- INSERT OR UPDATE（既存データなら更新、なければ挿入）
-- MySQL では INSERT ... ON DUPLICATE KEY UPDATE を使う
INSERT INTO students (id, name, score)
VALUES (1, '田中太郎', 95)
ON DUPLICATE KEY UPDATE score = 95;
-- id=1 が既存 → score だけ 95 に更新される
-- id=1 が存在しない → 新規挿入される
```

---

## 6. CRUD まとめ

| 操作 | SQL | 意味 |
|------|-----|------|
| **C**reate（作成） | `INSERT INTO ... VALUES ...` | データを追加 |
| **R**ead（読取） | `SELECT ... FROM ...` | データを取得 |
| **U**pdate（更新） | `UPDATE ... SET ... WHERE ...` | データを更新 |
| **D**elete（削除） | `DELETE FROM ... WHERE ...` | データを削除 |

> **CRUD = Create / Read / Update / Delete** はWebアプリの基本操作です。  
> Laravel の演習でも全く同じ概念が出てきます。
