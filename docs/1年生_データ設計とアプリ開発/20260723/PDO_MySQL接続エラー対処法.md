# PDO MySQL 拡張機能が有効化されない場合の対処法

## 対象のエラー

以下のようなエラーが出ている場合の対処法です。

```
Fatal error: Uncaught PDOException: could not find driver in C:\Users\nishida\Desktop\apple\5.php:2
Stack trace:
#0 C:\Users\nishida\Desktop\apple\5.php(2): PDO->__construct('mysql:host=127....', 'root', Object(SensitiveParameterValue))
#1 {main}
  thrown in C:\Users\nishida\Desktop\apple\5.php on line 2
```

### このエラーの意味

PHP から MySQL（データベース）に接続するとき、`PDO` という仕組みを使いますが、実際に MySQL と会話するための「ドライバー」という部品が別途必要です。

`could not find driver` は「そのドライバーが見つかりません」という意味のエラーです。つまり、**PHP が `pdo_mysql` という拡張機能（ドライバー）を読み込めていない**ために起きています。

原因は php.ini（PHP の設定ファイル）の設定にあることがほとんどです。以下の手順を上から順番に試してください。1つ試すごとに、必ず「サーバーの再起動」（[対策3](#対策3-サーバーを確実に再起動する)）を行ってからブラウザで確認してください。設定を変えただけではPHPに反映されません。

---

## 手順0: php.ini を開く

これから何度も php.ini というファイルを編集します。まずは開き方を確認しましょう。

1. エクスプローラー（フォルダ）や VS Code から `C:\php\php.ini` を開きます。
   - 見つからない場合は、ターミナルで `php --ini` と入力すると、使われている php.ini の場所が表示されます。
2. 以降、この php.ini の中身を編集していきます。

---

## 手順1: extension=pdo_mysql のコメントアウトを外す

php.ini の中には、たくさんの機能（拡張機能）のオン・オフを切り替える設定が並んでいます。行の先頭に `;`（セミコロン）が付いていると「その行は無効（コメントアウト）」という意味になります。

1. php.ini を開いた状態で `Ctrl + F`（検索）を押し、`extension=pdo_mysql` と入力して検索します。
2. 以下のように先頭に `;` が付いている行が見つかるはずです。

   ```ini
   ;extension=pdo_mysql
   ```

3. 行の先頭の `;` だけを削除します。

   ```ini
   extension=pdo_mysql
   ```

4. `Ctrl + S` で保存します。
5. [対策3](#対策3-サーバーを確実に再起動する) の手順でサーバーを再起動し、ブラウザで動作を確認します。

これで直ることが多いですが、直らない場合は次に進んでください。

---

## ここまでで解決しない場合

`;extension=pdo_mysql` のコメントアウトを外したのに解決しない場合、「拡張機能の実体ファイル（DLLファイル）が入っているフォルダの場所」を PHP が認識できていない可能性が高いです。

以下の対策を順番に確認してみてください。

## 対策1: extension_dir の設定を有効にする（★一番多い原因です）

`extension=pdo_mysql` はあくまで「pdo_mysql という機能を使う」という指示にすぎません。実際にその機能を実現する中身（DLLファイル）がどこにあるかを PHP に教える設定が `extension_dir` です。ここが正しく設定されていないと、指示だけあって中身が見つからない状態になります。

1. `C:\php\php.ini` を開きます。
2. キーボードで `Ctrl + F` を押し、`extension_dir` で検索します。
3. 以下のような行を探します（先頭に `;` が付いているはずです）。

   ```ini
   ;extension_dir = "ext"
   ```

4. 先頭の `;`（セミコロン）を消して有効化します。

   ```ini
   extension_dir = "ext"
   ```

5. `Ctrl + S` で保存し、[対策3](#対策3-サーバーを確実に再起動する) でサーバーを再起動して確認します。

   ※これでも解決しない場合は、`"ext"` という相対パスではなく、フルパス（絶対パス）で指定してみてください。

   ```ini
   extension_dir = "C:\php\ext"
   ```

   これは「`C:\php` フォルダの中の `ext` フォルダを見てね」と PHP に対して迷いなく伝える書き方です。書き換えたら、また保存してサーバーを再起動してください。

## 対策2: extension=mysqli も一緒に有効化してみる

`mysqli` も MySQL に接続するための別の拡張機能です。環境によっては、`pdo_mysql` が正常に動くために `mysqli` も一緒に有効になっている必要がある場合があります。念のため一緒に有効にしておきましょう。

1. php.ini で `extension=mysqli` を検索します。
2. 先頭に `;` があれば消します。

   ```ini
   extension=mysqli
   ```

3. `Ctrl + S` で保存し、サーバーを再起動して確認します。

## 対策3: サーバーを「確実に」再起動する

php.ini を書き換えても、今まさに動いている PHP サーバーには自動的に反映されません。設定ファイルは「サーバーを起動するとき」に一度だけ読み込まれる仕組みだからです。そのため、設定を変えたら必ずサーバーを一度完全に止めてから、もう一度起動し直す必要があります。

1. VS Code 下部のターミナル（黒い画面）を開きます。
2. ターミナルをクリックして `Ctrl + C` を数回押し、サーバーを完全に停止させます。ターミナルに `^C` と表示され、コマンドを入力できる状態（プロンプト）に戻れば停止できています。
3. もう一度、サーバーを起動するコマンドを入力します。

   ```bash
   php -S localhost:8000
   ```

4. ブラウザで対象のページ（例: `http://localhost:8000/5.php`）を再読み込み（更新）します。

---

## それでも解決しない場合の確認方法

「そもそも今 pdo_mysql が読み込まれているのか」を直接確認する方法があります。

1. ターミナルで以下のコマンドを実行します。

   ```bash
   php -m
   ```

   これは「現在 PHP に読み込まれている拡張機能の一覧」を表示するコマンドです。

2. 表示された一覧の中に `pdo_mysql` という文字があるか探してください。
   - **あれば** → 拡張機能自体は読み込めています。エラーが続く場合は、接続先の MySQL サーバーが起動しているか、接続情報（ホスト名・ユーザー名・パスワードなど）が正しいかを確認してください。
   - **なければ** → まだ拡張機能が読み込めていない状態です。上記の対策1〜3を、1つずつ確実に（保存＆サーバー再起動をセットで）やり直してみてください。
